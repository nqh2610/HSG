Bài Tập Tham Lam

Bộ đề rèn luyện kỹ năng Tham Lam (Greedy) dành cho HSG Tin học

Bài 1: XẾP SỮA
XEPSUA.INP / XEPSUA.OUT

Nông trại có N con bò và M chai sữa. Con bò thứ i chỉ chấp nhận sữa có giá không quá pi đồng. Chai sữa thứ j có giá cj đồng. Mỗi con bò uống tối đa 1 chai, mỗi chai cho tối đa 1 con bò. Tìm số con bò được uống sữa nhiều nhất.

Dữ liệu vào (XEPSUA.INP):
  • Dòng 1: Hai số nguyên N, M (1 ≤ N, M ≤ 105)
  • Dòng 2: N số nguyên p1, p2, ..., pN (1 ≤ pi ≤ 109)
  • Dòng 3: M số nguyên c1, c2, ..., cM (1 ≤ cj ≤ 109)
Kết quả ra (XEPSUA.OUT):
  • Một số nguyên duy nhất — số con bò được uống sữa tối đa

XEPSUA.INP

4 5
3 5 2 4
4 1 6 2 3

XEPSUA.OUT

4
📝 Code C++ — E1. Sort + Matching 
// Bài 1: XEPSUA — Greedy sort + two-pointer matching
#include <bits/stdc++.h>
using namespace std;

int main() {
    ifstream fin("XEPSUA.INP");
    ofstream fout("XEPSUA.OUT");
    int n, m;
    fin >> n >> m;
    vector<int> p(n), c(m);
    for (int i = 0; i < n; i++) fin >> p[i];
    for (int i = 0; i < m; i++) fin >> c[i];
    sort(p.begin(), p.end());
    sort(c.begin(), c.end());
    int ans = 0, j = 0;
    for (int i = 0; i < n && j < m; i++) {
        if (c[j] <= p[i]) {
            ans++;
            j++;
        }
    }
    fout << ans;
    return 0;
}
💡 Ý tưởng: Dùng Sắp xếp (A9) + Two Pointers (A5). Sắp xếp giá sữa tăng, giá chấp nhận của bò cũng tăng. Hai con trỏ: sữa rẻ nhất hiện tại và bò kén nhất hiện tại. Nếu sữa ≤ giá chấp nhận → ghép cặp, cả 2 tiến. Không → bò này yêu cầu quá rẻ không ai đáp ứng, tiến bò (vì sữa đã rẻ nhất rồi). Vì sao đúng? Cho bò rẻ sữa rẻ = không lãng phí sữa đắt cho bò dễ tính. O(N log N + M log M).
Bài 2: CHỌN ĐOẠN
CHONDOAN.INP / CHONDOAN.OUT

Có N hoạt động, hoạt động thứ i diễn ra từ thời điểm ai đến bi. Hai hoạt động chồng chéo nếu chúng có thời điểm chung (một hoạt động kết thúc đúng lúc hoạt động kia bắt đầu thì KHÔNG chồng chéo). Chọn nhiều hoạt động nhất sao cho không có hai hoạt động nào chồng chéo.

Dữ liệu vào (CHONDOAN.INP):
  • Dòng 1: Số nguyên N (1 ≤ N ≤ 105)
  • N dòng tiếp theo: Mỗi dòng hai số ai, bi (1 ≤ ai < bi ≤ 109)
Kết quả ra (CHONDOAN.OUT):
  • Một số nguyên duy nhất — số hoạt động tối đa chọn được

📝 Code C++ — E2. Activity Selection 
// Bài 2: CHONDOAN — Greedy sort by end time
#include <bits/stdc++.h>
using namespace std;

bool cmp(pair<int,int> a, pair<int,int> b) {
    return a.second < b.second;
}

int main() {
    ifstream fin("CHONDOAN.INP");
    ofstream fout("CHONDOAN.OUT");
    int n;
    fin >> n;
    vector<pair<int,int>> a(n);
    for (int i = 0; i < n; i++) fin >> a[i].first >> a[i].second;
    sort(a.begin(), a.end(), cmp);
    int ans = 0, last = 0;
    for (int i = 0; i < n; i++) {
        if (a[i].first >= last) {
            ans++;
            last = a[i].second;
        }
    }
    fout << ans;
    return 0;
}
💡 Ý tưởng: Dùng Tham lam — Activity Selection (B2). Sắp xếp hoạt động theo giờ KẾT THÚC tăng. Luôn chọn hoạt động kết thúc sớm nhất không giao với cái đã chọn. Vì sao đúng? Kết thúc sớm = giải phóng sân sớm = nhiều chỗ nhất cho hoạt động sau. Chọn hoạt động kết thúc muộn hơn chỉ có thể ít hơn hoặc bằng số hoạt động. O(N log N).
Bài 3: GHÉP SỐ
GHEPSO.INP / GHEPSO.OUT

Cho dãy N số nguyên dương (N chẵn). Ghép chúng thành N/2 cặp. Tổng của mỗi cặp là tổng hai số trong cặp. Hãy tìm cách ghép sao cho tổng lớn nhất trong các cặp là nhỏ nhất có thể.

Dữ liệu vào (GHEPSO.INP):
  • Dòng 1: Số nguyên N (2 ≤ N ≤ 105, N chẵn)
  • Dòng 2: N số nguyên a1, ..., aN (1 ≤ ai ≤ 106)
Kết quả ra (GHEPSO.OUT):
  • Một số nguyên duy nhất — giá trị tổng lớn nhất nhỏ nhất

GHEPSO.INP

6
1 3 5 2 8 4

GHEPSO.OUT

9
📝 Code C++ — E3. Pair Balance 
// Bài 3: GHEPSO — Sắp xếp, ghép nhỏ nhất với lớn nhất
#include <bits/stdc++.h>
using namespace std;

int main() {
    ifstream fin("GHEPSO.INP");
    ofstream fout("GHEPSO.OUT");
    int n;
    fin >> n;
    vector<int> a(n);
    for (int i = 0; i < n; i++) fin >> a[i];
    sort(a.begin(), a.end());
    int ans = 0;
    for (int i = 0; i < n / 2; i++)
        ans = max(ans, a[i] + a[n - 1 - i]);
    fout << ans;
    return 0;
}
💡 Ý tưởng: Dùng Sắp xếp (A9) + Tham lam (B2). Sắp xếp tăng. Ghép số nhỏ nhất với lớn nhất, nhỏ nhì với lớn nhì, ... VD: [1,2,3,4] → cặp (1,4)=5, (2,3)=5 → max cặp = 5. Nếu ghép khác: (1,2)=3, (3,4)=7 → max = 7, tệ hơn. Vì sao? Số lớn ghép số nhỏ = "bù trừ" lẫn nhau, giảm chênh lệch giữa các cặp → max nhỏ nhất. O(N log N).
Bài 4: THUYỀN BUỒM
THUYENBUOM.INP / THUYENBUOM.OUT

Có N người muốn qua sông. Mỗi thuyền chở tối đa 2 người và tổng trọng lượng không vượt quá C. Mỗi người có trọng lượng wi (đảm bảo wi ≤ C). Tìm số thuyền ít nhất để chở hết tất cả mọi người.

Dữ liệu vào (THUYENBUOM.INP):
  • Dòng 1: Hai số nguyên N, C (1 ≤ N ≤ 105; 1 ≤ C ≤ 109)
  • Dòng 2: N số nguyên w1, ..., wN (1 ≤ wi ≤ C)
Kết quả ra (THUYENBUOM.OUT):
  • Một số nguyên duy nhất — số thuyền ít nhất

40 60 80 50 30

📝 Code C++ — E4. Two-Pointer 
// Bài 4: THUYENBUOM — Greedy two-pointer
#include <bits/stdc++.h>
using namespace std;

int main() {
    ifstream fin("THUYENBUOM.INP");
    ofstream fout("THUYENBUOM.OUT");
    int n, C;
    fin >> n >> C;
    vector<int> w(n);
    for (int i = 0; i < n; i++) fin >> w[i];
    sort(w.begin(), w.end());
    int left = 0, right = n - 1, boats = 0;
    while (left <= right) {
        if (left == right) {
            boats++;
            break;
        }
        if (w[left] + w[right] <= C) left++;
        right--;
        boats++;
    }
    fout << boats;
    return 0;
}
💡 Ý tưởng: Dùng Two Pointers ngược chiều (A5) + Tham lam (B2). Sắp xếp trọng lượng tăng. left = nhẹ nhất, right = nặng nhất. Nếu left + right ≤ giới hạn → ghép cặp, cả 2 tiến (left++, right−−). Không vừa → người nặng đi riêng (right−−). Vì sao ghép nhẹ+nặng? Người nhẹ không ghép với ai khác tốt hơn được (ai cũng nặng hơn rồi), và người nặng cũng cần bạn nhẹ nhất để còn vừa thuyền. O(N log N).

// Bài 5: SANBAY — Sweep line (quét sự kiện)
#include <bits/stdc++.h>
using namespace std;

int main() {
    ifstream fin("SANBAY.INP");
    ofstream fout("SANBAY.OUT");
    int n;
    fin >> n;
    // Sự kiện: (thời gian, loại). Đến = +1, Đi = -1
    vector<pair<int,int>> ev;
    for (int i = 0; i < n; i++) {
        int a, d;
        fin >> a >> d;
        ev.push_back(make_pair(a, 1));
        ev.push_back(make_pair(d, -1));
    }
    // Sort: cùng thời điểm, xử lý đi (-1) trước đến (+1)
    sort(ev.begin(), ev.end());
    int cur = 0, ans = 0;
    for (int i = 0; i < (int)ev.size(); i++) {
        cur += ev[i].second;
        ans = max(ans, cur);
    }
    fout << ans;
    return 0;
}
💡 Ý tưởng: Dùng Sweep Line / Mảng hiệu (A3). Mỗi chuyến bay sinh 2 sự kiện: +1 khi đến (cần cổng), −1 khi đi (trả cổng). Sắp xếp sự kiện theo thời gian. Cùng thời điểm: xử lý đi trước (−1 < +1 nên pair tự sắp đúng). Quét từ trái sang, cộng dồn → số máy bay đang ở sân bay tại mỗi thời điểm. Max = số cổng cần. Vì sao xử lý đi trước? Vì máy bay đi giải phóng cổng trước khi máy bay đến dùng cổng. O(N log N).
Bài 6: BA LÔ PHÂN LƯỢNG
BALOPL.INP / BALOPL.OUT

📝 Code C++ — E6. Fractional Knapsack 
// Bài 6: BALOPL — Greedy sort by value/weight ratio
#include <bits/stdc++.h>
using namespace std;

struct Item { int w, v; };

bool cmp(Item a, Item b) {
    return (long long)a.v * b.w > (long long)b.v * a.w;
}

int main() {
    ifstream fin("BALOPL.INP");
    ofstream fout("BALOPL.OUT");
    int n, W;
    fin >> n >> W;
    vector<Item> items(n);
    for (int i = 0; i < n; i++) fin >> items[i].w >> items[i].v;
    sort(items.begin(), items.end(), cmp);
    double ans = 0;
    int rem = W;
    for (int i = 0; i < n && rem > 0; i++) {
        if (items[i].w <= rem) {
            ans += items[i].v;
            rem -= items[i].w;
        } else {
            ans += (double)items[i].v * rem / items[i].w;
            rem = 0;
        }
    }
    fout << fixed << setprecision(2) << ans;
    return 0;
}
💡 Ý tưởng: Dùng Tham lam — Fractional Knapsack (B2). Được lấy PHẦN (không bắt buộc lấy nguyên vật). Tính đơn giá = giá trị / khối lượng. Sắp xếp theo đơn giá giảm dần (để tránh sai số thực, so sánh bằng nhân chéo: v1×w2 với v2×w1). Lấy trọn vật đắt nhất trước, vật cuối lấy phần nếu không đủ chỗ. Vì sao tham lam đúng? Vì được cắt lẻ, 1 kg vật đắt luôn tốt hơn 1 kg vật rẻ. O(N log N).

// Bài 7: NOITEP — Min-heap, luôn nối 2 tệp nhỏ nhất
#include <bits/stdc++.h>
using namespace std;

int main() {
    ifstream fin("NOITEP.INP");
    ofstream fout("NOITEP.OUT");
    int n;
    fin >> n;
    priority_queue<long long, vector<long long>, greater<long long>> pq;
    for (int i = 0; i < n; i++) {
        long long x;
        fin >> x;
        pq.push(x);
    }
    long long ans = 0;
    while (pq.size() > 1) {
        long long a = pq.top(); pq.pop();
        long long b = pq.top(); pq.pop();
        ans += a + b;
        pq.push(a + b);
    }
    fout << ans;
    return 0;
}
💡 Ý tưởng: Dùng Tham lam (B2) + Priority Queue / Heap (C3). Dùng min-heap. Luôn lấy 2 tệp NHỎ NHẤT ra nối, kết quả đẩy lại heap. VD: [1,2,3,4] → nối 1+2=3 (chi phí 3), nối 3+3=6 (chi phí 6), nối 6+4=10 (chi phí 10), tổng = 19. Nếu nối lớn trước: 3+4=7, 7+2=9, 9+1=10, tổng = 26 — tệ hơn. Vì sao nhỏ trước? Tệp nhỏ ghép sớm sẽ bị cộng lại nhiều lần nhưng kích thước nhỏ nên ít tốn. O(N log N).

// Bài 8: NHIENLIEU — Đi xa nhất có thể, đổ khi cần
#include <bits/stdc++.h>
using namespace std;

int main() {
    ifstream fin("NHIENLIEU.INP");
    ofstream fout("NHIENLIEU.OUT");
    int L, C, n;
    fin >> L >> C >> n;
    vector<int> d(n + 2);
    d[0] = 0;
    for (int i = 1; i <= n; i++) fin >> d[i];
    d[n + 1] = L;
    // Kiểm tra khả thi: mọi khoảng cách liên tiếp ≤ C
    for (int i = 1; i <= n + 1; i++) {
        if (d[i] - d[i - 1] > C) {
            fout << -1;
            return 0;
        }
    }
    int fuel = C, stops = 0;
    for (int i = 1; i <= n + 1; i++) {
        fuel -= (d[i] - d[i - 1]);
        // Tại trạm i: nếu không đủ xăng đến trạm tiếp, đổ đầy
        if (i <= n && fuel < d[i + 1] - d[i]) {
            fuel = C;
            stops++;
        }
    }
    fout << stops;
    return 0;
}
💡 Ý tưởng: Dùng Tham lam (B2). Thêm điểm xuất phát (0) và đích (L). Duyệt từ trạm này sang trạm kế: trừ xăng. Tại mỗi trạm: chỉ đổ xăng khi không đủ đến trạm tiếp theo (đổ vừa đủ = đổ ít nhất). Nếu khoảng cách 2 trạm liên tiếp > C (dung tích bình) → không thể đến được. Vì sao đổ ít là tối ưu? Vì giá xăng các trạm như nhau, đổ ít lần = ít dừng. O(N).

📝 Code C++ — E9. BFS-like Jump 
// Bài 9: NHAYBAC — Greedy BFS-like (Jump Game II)
#include <bits/stdc++.h>
using namespace std;

int main() {
    ifstream fin("NHAYBAC.INP");
    ofstream fout("NHAYBAC.OUT");
    int n;
    fin >> n;
    vector<int> a(n);
    for (int i = 0; i < n; i++) fin >> a[i];
    if (n == 1) { fout << 0; return 0; }
    // curEnd: biên phải vùng đang xét, farthest: xa nhất có thể tới
    int jumps = 0, curEnd = 0, farthest = 0;
    for (int i = 0; i < n - 1; i++) {
        farthest = max(farthest, i + a[i]);
        if (i == curEnd) {
            if (farthest == curEnd) {
                fout << -1;
                return 0;
            }
            jumps++;
            curEnd = farthest;
            if (curEnd >= n - 1) break;
        }
    }
    fout << jumps;
    return 0;
}
💡 Ý tưởng: Dùng Tham lam (B2). Hình dung BFS theo tầng: mỗi bước nhảy = 1 tầng. Giữ 2 biến: curEnd = biên phải tầng hiện tại, farthest = xa nhất có thể đến từ tầng này. Duyệt i: cập nhật farthest = max(farthest, i + a[i]). Khi i đến curEnd: hết tầng, tăng bước nhảy, curEnd = farthest. Vì sao đúng? Trong cùng 1 bước nhảy, chọn điểm kế tiếp xa nhất = ít bước nhất. 📝 Code C++ — E10. Interval Cover 
// Bài 10: PHUDOAN — Sort by left, greedy extend
#include <bits/stdc++.h>
using namespace std;

bool cmp(pair<int,int> a, pair<int,int> b) {
    if (a.first != b.first) return a.first < b.first;
    return a.second > b.second;
}

int main() {
    ifstream fin("PHUDOAN.INP");
    ofstream fout("PHUDOAN.OUT");
    int A, B, n;
    fin >> A >> B >> n;
    vector<pair<int,int>> seg(n);
    for (int i = 0; i < n; i++) fin >> seg[i].first >> seg[i].second;
    sort(seg.begin(), seg.end(), cmp);
    int ans = 0, cur = A, idx = 0;
    while (cur < B) {
        int best = cur;
        while (idx < n && seg[idx].first <= cur) {
            best = max(best, seg[idx].second);
            idx++;
        }
        if (best == cur) {
            fout << -1;
            return 0;
        }
        cur = best;
        ans++;
    }
    fout << ans;
    return 0;
}
💡 Ý tưởng: Dùng Tham lam (B2) + Sắp xếp (A9). Sắp xếp đoạn theo đầu trái tăng. Giữ biến covered = điểm đã phủ đến. Tại mỗi bước: trong tất cả đoạn BẮt ĐẦU ≤ covered, chọn đoạn kéo dài xa nhất sang phải. Cập nhật covered = max_right. Vì sao chọn xa nhất? Vì đoạn phủ nhiều nhất = ít đoạn nhất. Nếu không tiến thêm được (covered không tăng) → không phủ kín. O(N log N).

📝 Code C++ — E11. Two-Pass Greedy 
// Bài 11: PHATKEO — Duyệt 2 lượt trái→phải, phải→trái
#include <bits/stdc++.h>
using namespace std;

int main() {
    ifstream fin("PHATKEO.INP");
    ofstream fout("PHATKEO.OUT");
    int n;
    fin >> n;
    vector<int> r(n);
    for (int i = 0; i < n; i++) fin >> r[i];
    vector<int> candy(n, 1);
    // Lượt 1: trái → phải
    for (int i = 1; i < n; i++)
        if (r[i] > r[i - 1]) candy[i] = candy[i - 1] + 1;
    // Lượt 2: phải → trái
    for (int i = n - 2; i >= 0; i--)
        if (r[i] > r[i + 1]) candy[i] = max(candy[i], candy[i + 1] + 1);
    long long ans = 0;
    for (int i = 0; i < n; i++) ans += candy[i];
    fout << ans;
    return 0;
}
💡 Ý tưởng: Dùng Tham lam 2 lượt (B2). Mỗi bạn vì cả 2 hướng (trái-phải). Lượt 1 (trái→phải): điểm[i] > điểm[i−1] thì kẹo[i] = kẹo[i−1]+1, đảm bảo ai giỏi hơn bên trái được nhiều hơn. Lượt 2 (phải→trái): tương tự cho hướng phải, lấy max với kết quả lượt 1 để đảm bảo cả 2 hướng. VD: [1,3,2] → lượt 1: [1,2,1], lượt 2: [1,2,1] → tổng = 4. O(N).

// Bài 12: SOTOTHAT — Sắp xếp bằng bộ so sánh tùy chỉnh
#include <bits/stdc++.h>
using namespace std;

bool cmp(string a, string b) {
    return a + b > b + a;
}

int main() {
    ifstream fin("SOTOTHAT.INP");
    ofstream fout("SOTOTHAT.OUT");
    int n;
    fin >> n;
    vector<string> a(n);
    for (int i = 0; i < n; i++) {
        int x;
        fin >> x;
        a[i] = to_string(x);
    }
    sort(a.begin(), a.end(), cmp);
    // Trường hợp toàn số 0
    if (a[0] == "0") {
        fout << "0";
        return 0;
    }
    string result;
    for (int i = 0; i < n; i++) result += a[i];
    fout << result;
    return 0;
}

📝 Code C++ — E13. Stack Greedy 
// Bài 13: XAUMIN — Ngăn xếp đơn điệu tăng
#include <bits/stdc++.h>
using namespace std;

int main() {
    ifstream fin("XAUMIN.INP");
    ofstream fout("XAUMIN.OUT");
    string s;
    int k;
    fin >> s >> k;
    string result;
    for (int i = 0; i < (int)s.size(); i++) {
        // Bỏ đỉnh stack nếu nó lớn hơn ký tự hiện tại và còn được xóa
        while (k > 0 && !result.empty() && result.back() > s[i]) {
            result.pop_back();
            k--;
        }
        result.push_back(s[i]);
    }
    // Nếu K còn dư, xóa từ cuối
    while (k > 0) { result.pop_back(); k--; }
    fout << result;
    return 0;
}
💡 Ý tưởng: Dùng Monotonic Stack (A7) + Tham lam (B2). Muốn số nhỏ nhất = các chữ số tăng dần từ trái sang (càng nhỏ càng tốt ở đầu). Dùng stack tăng: duyệt từng chữ số, nếu đỉnh stack LỚN hơn ký tự hiện tại và còn quota xóa → pop (xóa chữ số lớn ở vị trí cao để thay bằng số nhỏ hơn). Cuối cùng nếu K còn dư thì xóa từ cuối. VD: "1432" xóa 1 → xóa 4 (4>3) → "132". O(N).

📝 Code C++ — E14. Earliest Deadline First 
// Bài 14: TREHAN — Sắp xếp theo hạn chót (EDF)
#include <bits/stdc++.h>
using namespace std;

bool cmp(pair<int,int> a, pair<int,int> b) {
    return a.second < b.second;
}

int main() {
    ifstream fin("TREHAN.INP");
    ofstream fout("TREHAN.OUT");
    int n;
    fin >> n;
    vector<pair<int,int>> jobs(n); // (duration, deadline)
    for (int i = 0; i < n; i++) fin >> jobs[i].first >> jobs[i].second;
    sort(jobs.begin(), jobs.end(), cmp);
    long long time = 0, maxLate = 0;
    for (int i = 0; i < n; i++) {
        time += jobs[i].first;
        long long late = time - jobs[i].second;
        if (late > maxLate) maxLate = late;
    }
    fout << maxLate;
    return 0;
}
💡 Ý tưởng: Dùng Tham lam — Earliest Deadline First (B2). Sắp xếp theo hạn chót tăng dần. Làm việc tuần tự. Vì sao làm việc hạn sớm trước? Giả sử đổi 2 việc A (hạn sớm) và B (hạn muộn): A làm muộn hơn nhưng hạn sớm hơn → trễ A tăng. B làm sớm hơn nhưng hạn muộn hơn → trễ B giảm hoặc giữ. Độ trễ max tăng → đổi là tệ hơn → giữ nguyên là tối ưu. O(N log N).

📝 Code C++ — E15. DSU Scheduling 
// Bài 15: LICHLAMVIEC — Sort by profit + DSU tìm slot trống
#include <bits/stdc++.h>
using namespace std;

int par[100001];

int find(int x) {
    while (par[x] != x) {
        par[x] = par[par[x]]; // path halving
        x = par[x];
    }
    return x;
}

bool cmp(pair<int,int> a, pair<int,int> b) {
    return a.second > b.second;
}

int main() {
    ifstream fin("LICHLAMVIEC.INP");
    ofstream fout("LICHLAMVIEC.OUT");
    int n;
    fin >> n;
    vector<pair<int,int>> jobs(n); // (deadline, profit)
    int maxD = 0;
    for (int i = 0; i < n; i++) {
        fin >> jobs[i].first >> jobs[i].second;
        maxD = max(maxD, jobs[i].first);
    }
    for (int i = 0; i <= maxD; i++) par[i] = i;
    sort(jobs.begin(), jobs.end(), cmp);
    long long ans = 0;
    for (int i = 0; i < n; i++) {
        int slot = find(jobs[i].first);
        if (slot > 0) {
            ans += jobs[i].second;
            par[slot] = slot - 1;
        }
    }
    fout << ans;
    return 0;
}
💡 Ý tưởng: Dùng Tham lam (B2) + DSU (D3). Sắp xếp việc theo lợi nhuận GIẢM. Ưu tiên làm việc lời nhiều trước. Với mỗi việc (lời p, hạn d): tìm slot lớn nhất ≤ d còn trống để làm càng muộn càng tốt (dành slot sớm cho việc khác). Dùng DSU: find(d) trả về slot trống gần nhất ≤ d. Khi dùng slot s, hợp s với s−1 (lần sau find(s) sẽ trả về s−1). Vì sao DSU? Tìm slot trống thường O(d), DSU giảm xuống O(α(N)) gần như O(1). O(N log N).
