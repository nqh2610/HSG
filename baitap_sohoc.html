Bài Tập Số Học

Bộ đề rèn luyện kỹ năng Số học, SNT, Sàng, Lũy thừa nhanh, GCD/LCM dành cho HSG Tin học

Bài 1: ĐẾM SỐ NGUYÊN TỐ
DEMSNT.INP / DEMSNT.OUT

Cho T truy vấn, mỗi truy vấn gồm hai số nguyên L, R. Với mỗi truy vấn, đếm số lượng số nguyên tố trong đoạn [L, R].

Dữ liệu vào (DEMSNT.INP):
  • Dòng 1: T (1 ≤ T ≤ 105)
  • T dòng tiếp: L, R (1 ≤ L ≤ R ≤ 107)
Kết quả ra (DEMSNT.OUT):
  • T dòng, mỗi dòng một số nguyên — số SNT trong [L, R]

DEMSNT.INP

3
1 10
5 20
100 200

DEMSNT.OUT

4
6
21
Giải thích: [1,10] có 4 SNT: 2, 3, 5, 7.
📝 Code C++ — Sàng Eratosthenes + Prefix Sum 
#include <bits/stdc++.h>
using namespace std;
const int MAXN = 1e7;
bool notPrime[MAXN + 1];
int prefix[MAXN + 1];

int main() {
    ifstream in("DEMSNT.INP");
    ofstream out("DEMSNT.OUT");
    // Sàng Eratosthenes
    notPrime[0] = notPrime[1] = true;
    for (int i = 2; (long long)i * i <= MAXN; i++)
        if (!notPrime[i])
            for (int j = i * i; j <= MAXN; j += i)
                notPrime[j] = true;
    // Prefix sum: prefix[i] = số SNT trong [1, i]
    for (int i = 1; i <= MAXN; i++)
        prefix[i] = prefix[i - 1] + (!notPrime[i] ? 1 : 0);
    int T;
    in >> T;
    while (T--) {
        int L, R;
        in >> L >> R;
        out << prefix[R] - prefix[L - 1] << "\n";
    }
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Dùng Sàng Eratosthenes (E2) + Prefix Sum (A3). Sàng 1 lần đến 107 trong O(N log log N). Dựng mảng prefix[i] = số SNT ≤ i. Mỗi truy vấn trả lời O(1): prefix[R] − prefix[L−1]. Nếu kiểm tra từng số thì mỗi truy vấn O(R−L) × O(√R) sẽ TLE. Sàng + prefix đưa về O(1) mỗi truy vấn.
Bài 2: LŨY THỪA MODULO
LUYTHUA.INP / LUYTHUA.OUT

Cho T truy vấn, mỗi truy vấn gồm 3 số a, b, m. Tính ab mod m.

Dữ liệu vào (LUYTHUA.INP):
  • Dòng 1: T (1 ≤ T ≤ 105)
  • T dòng tiếp: a, b, m (1 ≤ a, m ≤ 109; 0 ≤ b ≤ 1018)
Kết quả ra (LUYTHUA.OUT):
  • T dòng, mỗi dòng kết quả ab mod m

LUYTHUA.INP

2
2 10 1000
3 100 1000000007

LUYTHUA.OUT

24
886041711
📝 Code C++ — Lũy thừa nhanh 
#include <bits/stdc++.h>
using namespace std;

long long power(long long a, long long b, long long m) {
    a %= m;
    long long res = 1;
    while (b > 0) {
        if (b & 1) res = res * a % m;
        a = a * a % m;
        b >>= 1;
    }
    return res;
}

int main() {
    ifstream in("LUYTHUA.INP");
    ofstream out("LUYTHUA.OUT");
    int T;
    in >> T;
    while (T--) {
        long long a, b, m;
        in >> a >> b >> m;
        out << power(a, b, m) << "\n";
    }
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Dùng Lũy thừa nhanh (E3). Nhân a lên b lần bình thường → O(b) = 1018 → TLE. Lũy thừa nhanh: chia b theo bit nhị phân. Mỗi bước: nếu bit cuối = 1 → nhân res với a; rồi a = a×a, b /= 2. Chỉ O(log b) phép nhân. Vì m ≤ 109, sau mod res và a đều < 109, tích < 1018 nằm trong long long. O(T × log b).
Bài 3: PHÂN TÍCH THỪA SỐ NGUYÊN TỐ
PHANTICH.INP / PHANTICH.OUT

Cho T số nguyên dương N. Với mỗi N, phân tích N thành tích các thừa số nguyên tố.

Dữ liệu vào (PHANTICH.INP):
  • Dòng 1: T (1 ≤ T ≤ 105)
  • T dòng tiếp: mỗi dòng 1 số N (2 ≤ N ≤ 107)
Kết quả ra (PHANTICH.OUT):
  • T dòng, mỗi dòng là phân tích dạng: p1^e1 * p2^e2 * ...

PHANTICH.INP

3
12
7
360

PHANTICH.OUT

2^2 * 3^1
7^1
2^3 * 3^2 * 5^1
📝 Code C++ — Sàng + Smallest Prime Factor 
#include <bits/stdc++.h>
using namespace std;
const int MAXN = 1e7;
int spf[MAXN + 1]; // smallest prime factor

void sieve() {
    for (int i = 2; i <= MAXN; i++) spf[i] = i;
    for (int i = 2; (long long)i * i <= MAXN; i++)
        if (spf[i] == i) // i là SNT
            for (int j = i * i; j <= MAXN; j += i)
                if (spf[j] == j) spf[j] = i;
}

int main() {
    ifstream in("PHANTICH.INP");
    ofstream out("PHANTICH.OUT");
    sieve();
    int T;
    in >> T;
    while (T--) {
        int n;
        in >> n;
        bool first = true;
        while (n > 1) {
            int p = spf[n], cnt = 0;
            while (n % p == 0) { n /= p; cnt++; }
            if (!first) out << " * ";
            out << p << "^" << cnt;
            first = false;
        }
        out << "\n";
    }
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Dùng Sàng biến thể (E2). Sàng thường chỉ trả lời "có/không phải SNT". Ở đây sàng thêm: spf[i] = ước nguyên tố nhỏ nhất của i. Khi phân tích: chia liên tục cho spf[n] cho đến khi n = 1. Mỗi lần chia: p = spf[n], đếm số lần chia hết. Nếu không sàng, phân tích mỗi số O(√N). T = 105 × √(107) ≈ 3×108 → sát TLE. Dùng spf: mỗi số phân tích O(log N) → tổng O(T log N).
Bài 4: TRUY VẤN GCD
GCDQUERY.INP / GCDQUERY.OUT

Cho mảng N số nguyên dương. Trả lời Q truy vấn, mỗi truy vấn L R: tính GCD(aL, aL+1, ..., aR).

Dữ liệu vào (GCDQUERY.INP):
  • Dòng 1: N, Q (1 ≤ N, Q ≤ 2×105)
  • Dòng 2: N số nguyên ai (1 ≤ ai ≤ 109)
  • Q dòng tiếp: L, R (1 ≤ L ≤ R ≤ N)
Kết quả ra (GCDQUERY.OUT):
  • Q dòng, mỗi dòng một số nguyên — GCD đoạn [L, R]

GCDQUERY.INP

5 3
12 6 18 3 9
1 3
2 4
1 5

GCDQUERY.OUT

6
3
3
📝 Code C++ — Sparse Table 
#include <bits/stdc++.h>
using namespace std;
const int MAXN = 2e5 + 5;
const int LOG = 18;
int sp[LOG][MAXN], _log[MAXN];

int main() {
    ifstream in("GCDQUERY.INP");
    ofstream out("GCDQUERY.OUT");
    int n, q;
    in >> n >> q;
    for (int i = 1; i <= n; i++) in >> sp[0][i];
    // Tiền xử lý log
    _log[1] = 0;
    for (int i = 2; i <= n; i++) _log[i] = _log[i / 2] + 1;
    // Xây Sparse Table
    for (int j = 1; j < LOG; j++)
        for (int i = 1; i + (1 << j) - 1 <= n; i++)
            sp[j][i] = __gcd(sp[j - 1][i], sp[j - 1][i + (1 << (j - 1))]);
    while (q--) {
        int l, r;
        in >> l >> r;
        int k = _log[r - l + 1];
        out << __gcd(sp[k][l], sp[k][r - (1 << k) + 1]) << "\n";
    }
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Dùng Sparse Table (C10) + GCD (E4). Truy vấn GCD đoạn [L, R] cần CTDL hỗ trợ. GCD có tính chất overlap-friendly: gcd(a, a) = a → dùng Sparse Table trả lời O(1) mỗi truy vấn. sp[j][i] = GCD đoạn bắt đầu i, dài 2j. Truy vấn: gcd(sp[k][L], sp[k][R−2k+1]) với k = log2(R−L+1). Dựng O(N log N), mỗi truy vấn O(1). Nếu duyệt thô → O(Q × N) ≈ 4×1010 → TLE.
Bài 5: HÀM EULER PHI
EULER.INP / EULER.OUT

Cho T số N. Với mỗi N, tính φ(N) — số lượng số nguyên trong [1, N] nguyên tố cùng nhau với N.

Dữ liệu vào (EULER.INP):
  • Dòng 1: T (1 ≤ T ≤ 105)
  • T dòng tiếp: mỗi dòng 1 số N (1 ≤ N ≤ 106)
Kết quả ra (EULER.OUT):
  • T dòng, mỗi dòng φ(N)

EULER.INP

3
1
6
12

EULER.OUT

1
2
4
Giải thích: φ(6) = 2 vì chỉ có 1 và 5 nguyên tố cùng nhau với 6.
📝 Code C++ — Sàng Euler Phi 
#include <bits/stdc++.h>
using namespace std;
const int MAXN = 1e6;
int phi_[MAXN + 1];

void sievePhi() {
    for (int i = 0; i <= MAXN; i++) phi_[i] = i;
    for (int i = 2; i <= MAXN; i++)
        if (phi_[i] == i) // i là SNT
            for (int j = i; j <= MAXN; j += i)
                phi_[j] -= phi_[j] / i;
}

int main() {
    ifstream in("EULER.INP");
    ofstream out("EULER.OUT");
    sievePhi();
    int T;
    in >> T;
    while (T--) {
        int n;
        in >> n;
        out << phi_[n] << "\n";
    }
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Dùng Sàng biến thể (E2). Công thức: φ(N) = N × ∏(1 − 1/p) với p là thừa số nguyên tố của N. Sàng giống Eratosthenes: khi tìm được SNT i, duyệt tất cả bội j của i, cập nhật phi[j] -= phi[j] / i. Sau sàng, tra bảng O(1). Nếu tính trực tiếp mỗi truy vấn: phân tích thừa số O(√N) × T = 105 × 103 = 108 → sát giới hạn. Sàng sẵn an toàn hơn.
Bài 6: NHÂN MA TRẬN
MATRIX.INP / MATRIX.OUT

Cho dãy Fibonacci: F(0) = 0, F(1) = 1, F(n) = F(n-1) + F(n-2). Tính F(N) mod 109+7.

Dữ liệu vào (MATRIX.INP):
  • Một số nguyên N (0 ≤ N ≤ 1018)
Kết quả ra (MATRIX.OUT):
  • Một số nguyên — F(N) mod 109+7

MATRIX.INP

10

MATRIX.OUT

55
📝 Code C++ — Nhân ma trận + Lũy thừa nhanh 
#include <bits/stdc++.h>
using namespace std;
const long long MOD = 1e9 + 7;
typedef vector<vector<long long>> Matrix;

Matrix mul(Matrix& A, Matrix& B) {
    int n = A.size();
    Matrix C(n, vector<long long>(n, 0));
    for (int i = 0; i < n; i++)
        for (int k = 0; k < n; k++)
            if (A[i][k])
                for (int j = 0; j < n; j++)
                    C[i][j] = (C[i][j] + A[i][k] * B[k][j]) % MOD;
    return C;
}

Matrix matpow(Matrix A, long long p) {
    int n = A.size();
    Matrix R(n, vector<long long>(n, 0));
    for (int i = 0; i < n; i++) R[i][i] = 1;
    while (p > 0) {
        if (p & 1) R = mul(R, A);
        A = mul(A, A);
        p >>= 1;
    }
    return R;
}

int main() {
    ifstream in("MATRIX.INP");
    ofstream out("MATRIX.OUT");
    long long n;
    in >> n;
    if (n == 0) { out << 0; in.close(); out.close(); return 0; }
    Matrix A = {{1, 1}, {1, 0}};
    Matrix R = matpow(A, n - 1);
    out << R[0][0];
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Dùng Lũy thừa nhanh ma trận (E3). Vòng lặp tính F(N) thường: O(N) = 1018 → TLE. Nhận xét: [F(n+1), F(n)] = [[1,1],[1,0]] × [F(n), F(n-1)]. Suy ra [F(n+1), F(n)] = An × [F(1), F(0)]. Tính An bằng lũy thừa nhanh: O(log N) lần nhân ma trận 2×2 → O(8 × log N) ≈ 500 phép nhân. Kỹ thuật này áp dụng cho mọi hệ thức truy hồi tuyến tính.
Bài 7: ĐẾM ƯỚC SỐ
DEMUOC.INP / DEMUOC.OUT

Cho mảng N số nguyên dương. Với mỗi phần tử ai, đếm số phần tử khác trong mảng là ước của ai.

Dữ liệu vào (DEMUOC.INP):
  • Dòng 1: N (1 ≤ N ≤ 105)
  • Dòng 2: N số nguyên ai (1 ≤ ai ≤ 106)
Kết quả ra (DEMUOC.OUT):
  • Một dòng N số nguyên, số thứ i là số ước của ai có trong mảng

DEMUOC.INP

5
1 2 3 6 12

DEMUOC.OUT

0 1 1 3 4
Giải thích: Ước của 12 có trong mảng: {1, 2, 3, 6} → 4 ước (không đếm chính nó).
📝 Code C++ — Đánh dấu bội 
#include <bits/stdc++.h>
using namespace std;
const int MAXV = 1e6 + 5;

int main() {
    ifstream in("DEMUOC.INP");
    ofstream out("DEMUOC.OUT");
    int n;
    in >> n;
    vector<int> a(n);
    static int cnt[MAXV] = {};
    static int divCnt[MAXV] = {};
    for (int i = 0; i < n; i++) {
        in >> a[i];
        cnt[a[i]]++;
    }
    // Với mỗi giá trị d có mặt, cộng 1 cho tất cả bội j của d
    for (int d = 1; d < MAXV; d++)
        if (cnt[d])
            for (int j = d; j < MAXV; j += d)
                divCnt[j] += cnt[d];
    for (int i = 0; i < n; i++)
        out << divCnt[a[i]] - 1 << " \n"[i == n - 1];
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Kỹ thuật giống Sàng (E2). Duyệt mỗi cặp (i,j) kiểm tra a[i] | a[j] → O(N2) TLE. Phương pháp: với mỗi giá trị d có trong mảng (cnt[d] > 0), duyệt tất cả bội j = d, 2d, 3d,...: divCnt[j] += cnt[d]. Tổng phép lặp ≈ maxV/1 + maxV/2 + ... ≈ maxV × ln(maxV) ≈ 107. Cuối cùng divCnt[a[i]] − 1 (trừ chính nó) là đáp án.
Bài 8: NGHỊCH ĐẢO MODULAR
MODULAR.INP / MODULAR.OUT

Cho T truy vấn, mỗi truy vấn gồm 2 số a, m (m là SNT, gcd(a,m)=1). Tìm x sao cho a×x ≡ 1 (mod m), 0 ≤ x < m.

Dữ liệu vào (MODULAR.INP):
  • Dòng 1: T (1 ≤ T ≤ 105)
  • T dòng tiếp: a, m (1 ≤ a < m ≤ 109; m là SNT)
Kết quả ra (MODULAR.OUT):
  • T dòng, mỗi dòng giá trị x

MODULAR.INP

2
3 7
2 1000000007

MODULAR.OUT

5
500000004
Giải thích: 3×5 = 15 ≡ 1 (mod 7).
📝 Code C++ — Fermat nhỏ + Lũy thừa nhanh 
#include <bits/stdc++.h>
using namespace std;

long long power(long long a, long long b, long long m) {
    a %= m;
    long long res = 1;
    while (b > 0) {
        if (b & 1) res = res * a % m;
        a = a * a % m;
        b >>= 1;
    }
    return res;
}

int main() {
    ifstream in("MODULAR.INP");
    ofstream out("MODULAR.OUT");
    int T;
    in >> T;
    while (T--) {
        long long a, m;
        in >> a >> m;
        // a^(-1) ≡ a^(m-2) (mod m), vì m nguyên tố (Fermat nhỏ)
        out << power(a, m - 2, m) << "\n";
    }
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Dùng Lũy thừa nhanh (E3) + Định lý Fermat nhỏ. Khi m là SNT: am−1 ≡ 1 (mod m) → a−1 ≡ am−2 (mod m). Chỉ cần tính am−2 mod m bằng lũy thừa nhanh → O(log m). Nghịch đảo modular là kỹ thuật nền tảng cho bài tổ hợp: C(n,k) mod p = n! × (k!)^{-1} × ((n-k)!)^{-1}.

  • T dòng tiếp: N, K (0 ≤ K ≤ N ≤ 106)

📝 Code C++ — Tiền xử lý giai thừa + nghịch đảo 
#include <bits/stdc++.h>
using namespace std;
const int MAXN = 1e6 + 5;
const long long MOD = 1e9 + 7;
long long fac[MAXN], inv_fac[MAXN];

long long power(long long a, long long b, long long m) {
    long long res = 1; a %= m;
    while (b > 0) {
        if (b & 1) res = res * a % m;
        a = a * a % m;
        b >>= 1;
    }
    return res;
}

void precompute() {
    fac[0] = 1;
    for (int i = 1; i < MAXN; i++) fac[i] = fac[i - 1] * i % MOD;
    inv_fac[MAXN - 1] = power(fac[MAXN - 1], MOD - 2, MOD);
    for (int i = MAXN - 2; i >= 0; i--)
        inv_fac[i] = inv_fac[i + 1] * (i + 1) % MOD;
}

long long C(int n, int k) {
    if (k < 0 || k > n) return 0;
    return fac[n] % MOD * inv_fac[k] % MOD * inv_fac[n - k] % MOD;
}

int main() {
    ifstream in("TOHOP.INP");
    ofstream out("TOHOP.OUT");
    precompute();
    int T;
    in >> T;
    while (T--) {
        int n, k;
        in >> n >> k;
        out << C(n, k) << "\n";
    }
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Dùng Lũy thừa nhanh (E3) + nghịch đảo modular. C(n,k) = n! / (k! × (n−k)!). Mod nguyên tố → dùng nghịch đảo: C(n,k) = fac[n] × inv_fac[k] × inv_fac[n−k]. Tiền xử lý: fac[i] = i! mod p tính tuyến tính. inv_fac[MAXN−1] = power(fac[MAXN−1], p−2, p), rồi tính ngược: inv_fac[i] = inv_fac[i+1] × (i+1). Xây O(N), mỗi truy vấn O(1). Nếu tính tam giác Pascal → O(N2) bộ nhớ và thời gian → TLE với N = 106.
Bài 10: BỘI CHUNG NHỎ NHẤT CỦA DÃY
BOICHUNG.INP / BOICHUNG.OUT

Cho dãy N số nguyên dương. Tính bội chung nhỏ nhất (LCM) của cả dãy. Nếu kết quả vượt 1018 thì in "OVERFLOW".

Dữ liệu vào (BOICHUNG.INP):
  • Dòng 1: N (1 ≤ N ≤ 105)
  • Dòng 2: N số nguyên ai (1 ≤ ai ≤ 109)
Kết quả ra (BOICHUNG.OUT):
  • Một số — LCM của dãy, hoặc "OVERFLOW" nếu vượt 1018

BOICHUNG.INP

3
4 6 10

BOICHUNG.OUT

60
Giải thích: LCM(4, 6) = 12. LCM(12, 10) = 60.
📝 Code C++ — GCD + LCM + Kiểm tra tràn 
#include <bits/stdc++.h>
using namespace std;
const long long LIMIT = 1e18;

int main() {
    ifstream in("BOICHUNG.INP");
    ofstream out("BOICHUNG.OUT");
    int n;
    in >> n;
    long long lcm = 1;
    bool overflow = false;
    for (int i = 0; i < n; i++) {
        long long a;
        in >> a;
        long long g = __gcd(lcm, a);
        long long d = a / g;
        // Kiểm tra lcm * d có tràn không
        if (d != 0 && lcm > LIMIT / d) {
            overflow = true;
            break;
        }
        lcm = lcm * d;
    }
    if (overflow) out << "OVERFLOW";
    else out << lcm;
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Dùng GCD (E4). LCM(a, b) = a / gcd(a, b) × b. Tính LCM lần lượt: lcm = LCM(lcm, a[i]). Mẹo chia trước nhân sau: tính d = a / gcd(lcm, a) rồi lcm = lcm × d → tránh tràn trung gian. Kiểm tra tràn: trước khi nhân, so lcm > 1018 / d. Chia trước nhân sau để tránh tràn trung gian, kiểm tra tràn trước khi nhân.
Bài 11: ĐẾM SỐ CHÍNH PHƯƠNG

Cho T truy vấn, mỗi truy vấn gồm hai số L, R. Đếm số lượng số chính phương trong đoạn [L, R].

Dữ liệu vào (CHINPHUONG.INP):
  • Dòng 1: T (1 ≤ T ≤ 105)
  • T dòng tiếp: L, R (1 ≤ L ≤ R ≤ 1018)
Kết quả ra (CHINPHUONG.OUT):
  • T dòng, mỗi dòng số lượng số chính phương trong [L, R]

1 10
4 16
1 1000000000000000000

CHINPHUONG.OUT

3
3
1000000000
Giải thích: [1,10]: 1, 4, 9 → 3 số. [4,16]: 4, 9, 16 → 3 số.
📝 Code C++ — Công thức toán + sqrtl 
#include <bits/stdc++.h>
using namespace std;

// Đếm số chính phương trong [1, n]
long long cntSq(long long n) {
    if (n <= 0) return 0;
    long long s = (long long)sqrtl((long double)n);
    // Hiệu chỉnh sai số làm tròn
    while (s * s > n) s--;
    while ((s + 1) * (s + 1) <= n) s++;
    return s;
}

int main() {
    ifstream in("CHINPHUONG.INP");
    ofstream out("CHINPHUONG.OUT");
    int t;
    in >> t;
    while (t--) {
        long long l, r;
        in >> l >> r;
        out << cntSq(r) - cntSq(l - 1) << "\n";
    }
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Số chính phương trong [1, N] = ⌊√N⌋. Đếm trong [L, R] = cntSq(R) − cntSq(L−1). Duyệt brute từ L→R → TLE với R ≤ 1018. Dùng sqrtl + hiệu chỉnh: sau khi tính s = ⌊√N⌋, kiểm tra s×s ≤ N và (s+1)² > N để tránh sai số floating-point. Mỗi truy vấn O(1). Kỹ thuật hiệu chỉnh sqrt rất quan trọng khi làm việc với số lớn.

Dữ liệu vào (HOANHAO.INP):
  • Một số nguyên N (2 ≤ N ≤ 1012)
Kết quả ra (HOANHAO.OUT):
  • Dòng 1: "YES" hoặc "NO"
  • Dòng 2: nếu YES → các ước thực sự tăng dần (cách nhau dấu cách); nếu NO → tổng các ước thực sự

HOANHAO.OUT

YES
1 2 4 7 14
Giải thích: Ước thực sự của 28: 1, 2, 4, 7, 14. Tổng = 1+2+4+7+14 = 28 = N → hoàn hảo.
📝 Code C++ — Liệt kê ước O(√N) 
#include <bits/stdc++.h>
using namespace std;

int main() {
    ifstream in("HOANHAO.INP");
    ofstream out("HOANHAO.OUT");
    long long n;
    in >> n;
    vector<long long> divs;
    long long sum = 0;
    for (long long i = 1; i * i <= n; i++) {
        if (n % i == 0) {
            if (i < n) { divs.push_back(i); sum += i; }
            if (i != n / i && n / i < n) {
                divs.push_back(n / i);
                sum += n / i;
            }
        }
    }
    sort(divs.begin(), divs.end());
    if (sum == n) {
        out << "YES" << "\n";
        for (int i = 0; i < (int)divs.size(); i++)
            out << divs[i] << " \n"[i + 1 == (int)divs.size()];
    } else {
        out << "NO" << "\n" << sum;
    }
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Duyệt i từ 1 đến √N. Nếu N chia hết cho i → thêm cả i và N/i vào danh sách ước (loại N). Duyệt đến N → O(N) ≈ 1012 → TLE. Duyệt đến √N → O(√N) ≈ 106 → nhanh. So tổng ước với N. Kỹ thuật liệt kê ước O(√N) là nền tảng cho mọi bài số học.
Bài 13: FIBONACCI MODULO

  • Dòng 1: T (1 ≤ T ≤ 105)
  • T dòng tiếp: N (0 ≤ N ≤ 1018)
Kết quả ra (FIBONACCI.OUT):
  • T dòng, mỗi dòng F(N) mod 109+7

50
1000000000000000000

FIBONACCI.OUT

55
586268941
209783453
📝 Code C++ — Nhân ma trận nhanh 
#include <bits/stdc++.h>
using namespace std;
const long long MOD = 1e9 + 7;
typedef vector<vector<long long>> Matrix;

Matrix mul(const Matrix &A, const Matrix &B) {
    int n = A.size();
    Matrix C(n, vector<long long>(n, 0));
    for (int i = 0; i < n; i++)
        for (int k = 0; k < n; k++)
            if (A[i][k])
                for (int j = 0; j < n; j++)
                    C[i][j] = (C[i][j] + A[i][k] * B[k][j]) % MOD;
    return C;
}

Matrix power(Matrix A, long long p) {
    int n = A.size();
    Matrix R(n, vector<long long>(n, 0));
    for (int i = 0; i < n; i++) R[i][i] = 1;
    for (; p; p >>= 1) {
        if (p & 1) R = mul(R, A);
        A = mul(A, A);
    }
    return R;
}

int main() {
    ifstream in("FIBONACCI.INP");
    ofstream out("FIBONACCI.OUT");
    int t;
    in >> t;
    while (t--) {
        long long n;
        in >> n;
        if (n <= 1) { out << n << "\n"; continue; }
        Matrix A = {{1, 1}, {1, 0}};
        Matrix R = power(A, n - 1);
        out << R[0][0] << "\n";
    }
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Dùng Nhân ma trận nhanh (E2). Tính F(N) bằng vòng lặp → O(N) ≈ 1018 → TLE. Ma trận: [F(n+1), F(n)] = [[1,1],[1,0]]n × [F(1), F(0)]. Lũy thừa nhanh ma trận 2×2 → O(8 × log N) ≈ 500 phép nhân. Mỗi truy vấn O(log N).

📝 Code C++ — Sàng + Phân loại 
#include <bits/stdc++.h>
using namespace std;
const int MAXV = 1e7 + 1;
bool is_prime[MAXV];

void sieve() {
    fill(is_prime, is_prime + MAXV, true);
    is_prime[0] = is_prime[1] = false;
    for (int i = 2; (long long)i * i < MAXV; i++)
        if (is_prime[i])
            for (int j = i * i; j < MAXV; j += i)
                is_prime[j] = false;
}

int main() {
    ifstream in("SOCHAN.INP");
    ofstream out("SOCHAN.OUT");
    sieve();
    int n;
    in >> n;
    int even = 0, odd = 0;
    vector<int> primes;
    for (int i = 0; i < n; i++) {
        int a;
        in >> a;
        if (a % 2 == 0) even++;
        else odd++;
        if (is_prime[a]) primes.push_back(a);
    }
    out << even << "\n" << odd << "\n" << primes.size() << "\n";
    for (int i = 0; i < (int)primes.size(); i++)
        out << primes[i] << " \n"[i + 1 == (int)primes.size()];
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Dùng Sàng Eratosthenes (E1). Kiểm tra SNT cho từng a[i] bằng trial division → O(√a) mỗi số → chậm nếu N lớn. Sàng 1 lần đến 107 → tra bảng O(1) mỗi số. Chẵn/lẻ kiểm tra bit cuối (a % 2). Tổng O(maxV log log maxV + N).

#include <bits/stdc++.h>
using namespace std;

int main() {
    ifstream in("USCLN.INP");
    ofstream out("USCLN.OUT");
    int t;
    in >> t;
    while (t--) {
        long long a, b;
        in >> a >> b;
        long long g = __gcd(a, b);

Bài 16: ĐỔI CƠ SỐ
HEDEM.INP / HEDEM.OUT

Cho số nguyên N ở hệ cơ số A. Chuyển N sang hệ cơ số B. Các chữ số lớn hơn 9 dùng ký tự A, B, C, … (ví dụ 10→A, 15→F).

Dữ liệu vào (HEDEM.INP):
  • Dòng 1: Hai số A, B (2 ≤ A, B ≤ 16)
  • Dòng 2: Xâu biểu diễn N ở cơ số A (độ dài ≤ 60)
Kết quả ra (HEDEM.OUT):
  • Xâu biểu diễn N ở cơ số B

HEDEM.INP

10 2
255

HEDEM.OUT

11111111
Giải thích: 255 (cơ số 10) = 11111111 (cơ số 2).
📝 Code C++ — Đổi cơ số qua trung gian 
#include <bits/stdc++.h>
using namespace std;

int charToVal(char c) {
    if (c >= '0' && c <= '9') return c - '0';
    return c - 'A' + 10;
}
char valToChar(int v) {
    if (v < 10) return '0' + v;
    return 'A' + v - 10;
}

int main() {
    ifstream in("HEDEM.INP");
    ofstream out("HEDEM.OUT");
    int a, b;
    string s;
    in >> a >> b >> s;
    // Bước 1: Cơ số A → số thập phân (long long)
    long long val = 0;
    for (char c : s)
        val = val * a + charToVal(c);
    // Bước 2: Số thập phân → cơ số B
    if (val == 0) { out << 0; in.close(); out.close(); return 0; }
    string res;
    while (val > 0) {
        res += valToChar(val % b);
        val /= b;
    }
    reverse(res.begin(), res.end());
    out << res;
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Đổi cơ số qua 2 bước: (1) cơ số A → giá trị thập phân bằng Horner: val = val × A + digit, (2) thập phân → cơ số B bằng chia lấy dư liên tục: lấy val % B rồi val /= B, đảo ngược xâu kết quả. Chữ số > 9 dùng ký tự A-F. O(len × log val).

📝 Code C++ — Duyệt + Kiểm tra 
#include <bits/stdc++.h>
using namespace std;

bool isPalin(int n) {
    string s = to_string(n);
    int l = 0, r = s.size() - 1;
    while (l < r)
        if (s[l++] != s[r--]) return false;
    return true;
}

int digitSum(int n) {
    int s = 0;
    while (n) { s += n % 10; n /= 10; }
    return s;
}

int main() {
    ifstream in("PALINDROME.INP");
    ofstream out("PALINDROME.OUT");
    int l, r, k;
    in >> l >> r >> k;
    vector<int> res;
    for (int i = l; i <= r; i++)
        if (isPalin(i) && digitSum(i) % k == 0)
            res.push_back(i);
    out << res.size() << "\n";
    for (int i = 0; i < (int)res.size(); i++)
        out << res[i] << " \n"[i + 1 == (int)res.size()];
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Kiểm tra palindrome: chuyển thành xâu, so sánh 2 đầu tiến vào giữa. Tổng chữ số: lấy dư 10 liên tục. R ≤ 106 → duyệt trực tiếp O(R × số chữ số) là đủ nhanh. Bài kết hợp 2 kỹ thuật xử lý chữ số cơ bản.
Bài 18: SỐ ARMSTRONG

Cho hai số L, R. Liệt kê tất cả số Armstrong trong đoạn [L, R]. Số Armstrong là số có k chữ số mà tổng lũy thừa bậc k của từng chữ số bằng chính nó (ví dụ 153 = 13 + 53 + 33).

Dữ liệu vào (ARMSTRONG.INP):
  • Một dòng: L, R (1 ≤ L ≤ R ≤ 109)
Kết quả ra (ARMSTRONG.OUT):
  • Dòng 1: Số lượng số Armstrong trong [L, R]
  • Dòng 2: Liệt kê các số đó

ARMSTRONG.INP

1 1000

ARMSTRONG.OUT

13
1 2 3 4 5 6 7 8 9 153 370 371 407
Giải thích: 153 = 1³+5³+3³. 370 = 3³+7³+0³. 371 = 3³+7³+1³. 407 = 4³+0³+7³.
📝 Code C++ — Tiền xử lý lũy thừa 
#include <bits/stdc++.h>
using namespace std;

// Chỉ có 88 số Armstrong ≤ 10^18, sinh trước rồi lọc
long long pw[10][11]; // pw[d][k] = d^k

int numDigits(long long n) {
    int k = 0;
    while (n) { k++; n /= 10; }
    return k;
}

bool isArmstrong(long long n) {
    int k = numDigits(n);
    long long sum = 0, tmp = n;
    while (tmp) {
        int d = tmp % 10;
        sum += pw[d][k];
        if (sum > n) return false; // cắt sớm
        tmp /= 10;
    }
    return sum == n;
}

int main() {
    ifstream in("ARMSTRONG.INP");
    ofstream out("ARMSTRONG.OUT");
    // Tiền xử lý lũy thừa
    for (int d = 0; d <= 9; d++) {
        pw[d][0] = 1;
        for (int k = 1; k <= 10; k++)
            pw[d][k] = pw[d][k - 1] * d;
    }
    // Sinh tất cả số Armstrong ≤ 10^9
    // Chỉ có rất ít (~30 số), duyệt brute từng số chữ số
    vector<long long> all;
    for (int k = 1; k <= 10; k++) {
        long long lo = (k == 1) ? 1 : 1;
        for (long long t = lo; t <= 9; t++) lo *= 10;
        // Dùng cách khác: duyệt tổ hợp chữ số
    }
    // Cách đơn giản nhất cho R ≤ 10^9: liệt kê sẵn
    long long known[] = {1,2,3,4,5,6,7,8,9,
        153,370,371,407,
        1634,8208,9474,
        54748,92727,93084,
        548834,
        1741725,4210818,9800817,9926315,
        24678050,24678051,88593477,
        146511208,472335975,534494836,912985153};
    long long l, r;
    in >> l >> r;
    vector<long long> res;
    for (long long x : known)
        if (x >= l && x <= r) res.push_back(x);
    out << res.size() << "\n";
    for (int i = 0; i < (int)res.size(); i++)
        out << res[i] << " \n"[i + 1 == (int)res.size()];
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Duyệt từ L→R kiểm tra từng số → O(R × k) ≈ 1010 → TLE. Nhận xét: số Armstrong ≤ 109 chỉ có 31 số (đã biết trước). Có thể liệt kê sẵn hoặc sinh bằng tổ hợp chữ số: chọn k chữ số (lặp), tính tổng dk, kiểm tra kết quả có đúng k chữ số không. Tiền xử lý pw[d][k] = dk để tránh tính lại. Bài rèn kỹ năng phân tích bài toán: thay vì brute force, nhận ra tính chất đặc biệt của tập nghiệm.
Bài 19: GIẢ THUYẾT GOLDBACH

Dữ liệu vào (GOLDBACH.INP):
  • Dòng 1: T (1 ≤ T ≤ 105)
  • T dòng tiếp: N chẵn (4 ≤ N ≤ 107)
Kết quả ra (GOLDBACH.OUT):
  • T dòng, mỗi dòng p và q

GOLDBACH.INP

4
20
100

GOLDBACH.OUT

3 17
3 97
Giải thích: 4 = 2+2. 20 = 3+17 (p nhỏ nhất). 100 = 3+97.
📝 Code C++ — Sàng + Duyệt 
#include <bits/stdc++.h>
using namespace std;
const int MAXN = 1e7 + 1;
bool is_prime[MAXN];

void sieve() {
    fill(is_prime, is_prime + MAXN, true);
    is_prime[0] = is_prime[1] = false;
    for (int i = 2; (long long)i * i < MAXN; i++)
        if (is_prime[i])
            for (int j = i * i; j < MAXN; j += i)
                is_prime[j] = false;
}

int main() {
    ifstream in("GOLDBACH.INP");
    ofstream out("GOLDBACH.OUT");
    sieve();
    int t;
    in >> t;
    while (t--) {
        int n;
        in >> n;
        for (int p = 2; p <= n / 2; p++)
            if (is_prime[p] && is_prime[n - p]) {
                out << p << " " << n - p << "\n";
                break;
            }
    }
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Dùng Sàng Eratosthenes (E1). Sàng 1 lần đến 107. Với mỗi N, duyệt p từ 2 tăng dần: nếu p và N−p đều nguyên tố → in ra và dừng (p nhỏ nhất). Không sàng trước → phải kiểm tra SNT bằng trial division O(√N) mỗi lần → chậm hơn. Thực tế p rất nhỏ (thường ≤ vài chục), nên mỗi truy vấn gần như O(1) nhờ tra bảng sàng.
