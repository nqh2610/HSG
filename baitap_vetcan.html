Bài Tập Vét Cạn & Quay Lui

Bộ đề rèn luyện kỹ năng Vét cạn, Quay lui, Bitmask, Nhánh cận dành cho HSG Tin học

Bài 1: TẬP CON CÓ TỔNG K
TAPCON.INP / TAPCON.OUT

Cho mảng gồm N số nguyên dương a1, a2, ..., aN. Đếm số tập con (có thể rỗng) có tổng bằng K.

Dữ liệu vào (TAPCON.INP):
  • Dòng 1: Hai số nguyên N, K (1 ≤ N ≤ 20; 1 ≤ K ≤ 109)
  • Dòng 2: N số nguyên ai (1 ≤ ai ≤ 109)
Kết quả ra (TAPCON.OUT):
  • Một số nguyên duy nhất — số tập con có tổng bằng K

TAPCON.INP

4 5
1 2 3 4

TAPCON.OUT

2
Giải thích: Các tập con có tổng = 5: {1, 4} và {2, 3}.
📝 Code C++ — Bitmask duyệt 2^N 
#include <bits/stdc++.h>
using namespace std;

int main() {
    ifstream in("TAPCON.INP");
    ofstream out("TAPCON.OUT");
    int n; long long k;
    in >> n >> k;
    vector<long long> a(n);
    for (int i = 0; i < n; i++) in >> a[i];
    int ans = 0;
    for (int mask = 0; mask < (1 << n); mask++) {
        long long sum = 0;
        for (int i = 0; i < n; i++)
            if (mask >> i & 1) sum += a[i];
        if (sum == k) ans++;
    }
    out << ans;
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Dùng Bitmask duyệt tập con (B1d2). N ≤ 20 → 220 ≈ 106 — vừa đủ. Mỗi bit i trong mask = 1 nghĩa là chọn a[i]. Duyệt tất cả mask từ 0 đến 2N−1, tính tổng phần tử được chọn, đếm mask có tổng = K. O(N × 2N).
Bài 2: HOÁN VỊ ĐẸP
HOANVI.INP / HOANVI.OUT

Một hoán vị (p1, p2, ..., pN) của (1, 2, ..., N) được gọi là "đẹp" nếu |pi − i| ≤ 1 với mọi i. Đếm số hoán vị đẹp của N phần tử.

Dữ liệu vào (HOANVI.INP):
  • Một số nguyên N (1 ≤ N ≤ 15)
Kết quả ra (HOANVI.OUT):
  • Một số nguyên duy nhất — số hoán vị đẹp

HOANVI.INP

3

HOANVI.OUT

3
Giải thích: 3 hoán vị đẹp: (1,2,3), (1,3,2), (2,1,3).
📝 Code C++ — Quay lui + cắt nhánh 
#include <bits/stdc++.h>
using namespace std;
int n, ans = 0;
bool used[20];

void solve(int pos) {
    if (pos > n) { ans++; return; }
    for (int v = max(1, pos - 1); v <= min(n, pos + 1); v++) {
        if (!used[v]) {
            used[v] = true;
            solve(pos + 1);
            used[v] = false;
        }
    }
}

int main() {
    ifstream in("HOANVI.INP");
    ofstream out("HOANVI.OUT");
    in >> n;
    solve(1);
    out << ans;
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Dùng Quay lui + cắt nhánh (B1d4). Tại vị trí pos, chỉ thử giá trị v ∈ {pos−1, pos, pos+1} (vì |v − pos| ≤ 1). Mỗi vị trí chỉ thử tối đa 3 giá trị thay vì N → cắt nhánh rất mạnh. Mảng used[] đánh dấu giá trị đã dùng. N ≤ 15 nhưng nhờ cắt nhánh, thực tế chạy rất nhanh (tối đa ~3N nhánh, nhưng hầu hết bị cắt sớm).
Bài 3: CHIA KẸO
CHIAKEO.INP / CHIAKEO.OUT

Có N gói kẹo, gói thứ i có ai viên. Chia N gói thành 2 phần sao cho chênh lệch số kẹo giữa 2 phần là nhỏ nhất. In chênh lệch nhỏ nhất.

Dữ liệu vào (CHIAKEO.INP):
  • Dòng 1: Số nguyên N (1 ≤ N ≤ 20)
  • Dòng 2: N số nguyên ai (1 ≤ ai ≤ 109)
Kết quả ra (CHIAKEO.OUT):
  • Một số nguyên duy nhất — chênh lệch nhỏ nhất

CHIAKEO.INP

4
3 1 4 2

CHIAKEO.OUT

0
Giải thích: Chia {3, 2} và {4, 1} → tổng đều = 5, chênh lệch = 0.
📝 Code C++ — Bitmask duyệt tập con 
#include <bits/stdc++.h>
using namespace std;

int main() {
    ifstream in("CHIAKEO.INP");
    ofstream out("CHIAKEO.OUT");
    int n;
    in >> n;
    vector<long long> a(n);
    long long total = 0;
    for (int i = 0; i < n; i++) { in >> a[i]; total += a[i]; }
    long long ans = total;
    for (int mask = 0; mask < (1 << n); mask++) {
        long long s = 0;
        for (int i = 0; i < n; i++)
            if (mask >> i & 1) s += a[i];
        ans = min(ans, abs(total - 2 * s));
    }
    out << ans;
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Dùng Bitmask (B1d2). Chia 2 phần = chọn 1 tập con S, phần còn lại tự động. Chênh lệch = |total − 2×sum(S)|. Duyệt 2N tập con, tìm min chênh lệch. N ≤ 20 → 220 ≈ 106. O(N × 2N).
Bài 4: XẾP HẬU
XEPHAU.INP / XEPHAU.OUT

Đặt N quân hậu lên bàn cờ N×N sao cho không có 2 quân hậu nào tấn công nhau (cùng hàng, cùng cột, hoặc cùng đường chéo). Đếm số cách đặt.

Dữ liệu vào (XEPHAU.INP):
  • Một số nguyên N (1 ≤ N ≤ 14)
Kết quả ra (XEPHAU.OUT):
  • Một số nguyên duy nhất — số cách đặt hợp lệ

XEPHAU.INP

4

XEPHAU.OUT

2
📝 Code C++ — Backtracking cắt nhánh 
#include <bits/stdc++.h>
using namespace std;
int n, ans = 0;
bool col[15], d1[30], d2[30];

void solve(int row) {
    if (row == n) { ans++; return; }
    for (int c = 0; c < n; c++) {
        if (!col[c] && !d1[row - c + n] && !d2[row + c]) {
            col[c] = d1[row - c + n] = d2[row + c] = true;
            solve(row + 1);
            col[c] = d1[row - c + n] = d2[row + c] = false;
        }
    }
}

int main() {
    ifstream in("XEPHAU.INP");
    ofstream out("XEPHAU.OUT");
    in >> n;
    solve(0);
    out << ans;
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Dùng Quay lui (B1d4). Đặt từng hàng: hàng row thử cột c. Điều kiện: cột c trống, 2 đường chéo (row−c) và (row+c) trống. Mảng col[], d1[], d2[] đánh dấu cột và 2 đường chéo. Mỗi hàng thử N cột nhưng nhờ cắt sớm bằng 3 điều kiện, nhánh bị loại rất nhiều. N ≤ 14 chạy dưới 1 giây.
Bài 5: GẶP NHAU Ở GIỮA
GAPMID.INP / GAPMID.OUT

Cho N số nguyên a1, a2, ..., aN. Đếm số tập con có tổng bằng K. (Bài giống Bài 1 nhưng N lớn hơn.)

Dữ liệu vào (GAPMID.INP):
  • Dòng 1: Hai số nguyên N, K (1 ≤ N ≤ 40; 1 ≤ K ≤ 1018)
  • Dòng 2: N số nguyên ai (1 ≤ ai ≤ 1015)
Kết quả ra (GAPMID.OUT):
  • Một số nguyên duy nhất — số tập con có tổng bằng K

GAPMID.INP

4 5
1 2 3 4

GAPMID.OUT

2
📝 Code C++ — Meet in the Middle 
#include <bits/stdc++.h>
using namespace std;

int main() {
    ifstream in("GAPMID.INP");
    ofstream out("GAPMID.OUT");
    int n; long long k;
    in >> n >> k;
    vector<long long> a(n);
    for (int i = 0; i < n; i++) in >> a[i];
    int h1 = n / 2, h2 = n - h1;
    // Sinh tổng nửa trái
    vector<long long> left;
    for (int mask = 0; mask < (1 << h1); mask++) {
        long long s = 0;
        for (int i = 0; i < h1; i++)
            if (mask >> i & 1) s += a[i];
        left.push_back(s);
    }
    sort(left.begin(), left.end());
    // Sinh tổng nửa phải, ghép
    long long ans = 0;
    for (int mask = 0; mask < (1 << h2); mask++) {
        long long s = 0;
        for (int i = 0; i < h2; i++)
            if (mask >> i & 1) s += a[h1 + i];
        long long need = k - s;
        ans += upper_bound(left.begin(), left.end(), need)
             - lower_bound(left.begin(), left.end(), need);
    }
    out << ans;
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Dùng Meet in the Middle (B1d5). N ≤ 40 → 240 quá lớn. Chia đôi: nửa trái N/2 phần tử, nửa phải N/2 phần tử. Mỗi nửa duyệt 220 ≈ 106. Nửa trái: sinh tất cả tổng tập con, sort. Nửa phải: với mỗi tổng s, tìm need = K − s trong nửa trái bằng binary search (upper_bound − lower_bound = đếm số lần xuất hiện). O(2N/2 × N).
Bài 6: SUDOKU NHỎ
SUDOKU.INP / SUDOKU.OUT

Cho bảng Sudoku 9×9 với một số ô đã điền (1-9), các ô trống = 0. Điền các ô trống sao cho mỗi hàng, mỗi cột, mỗi ô 3×3 đều chứa đủ 1-9. In bảng hoàn chỉnh (đảm bảo có đáp án duy nhất).

Dữ liệu vào (SUDOKU.INP):
  • 9 dòng, mỗi dòng 9 số nguyên (0 = ô trống)
Kết quả ra (SUDOKU.OUT):
  • 9 dòng, mỗi dòng 9 số — bảng đã điền đầy đủ

SUDOKU.INP

5 3 0 0 7 0 0 0 0
6 0 0 1 9 5 0 0 0
0 9 8 0 0 0 0 6 0
8 0 0 0 6 0 0 0 3
4 0 0 8 0 3 0 0 1
7 0 0 0 2 0 0 0 6
0 6 0 0 0 0 2 8 0
0 0 0 4 1 9 0 0 5
0 0 0 0 8 0 0 7 9

SUDOKU.OUT

5 3 4 6 7 8 9 1 2
6 7 2 1 9 5 3 4 8
1 9 8 3 4 2 5 6 7
8 5 9 7 6 1 4 2 3
4 2 6 8 5 3 7 9 1
7 1 3 9 2 4 8 5 6
9 6 1 5 3 7 2 8 4
2 8 7 4 1 9 6 3 5
3 4 5 2 8 6 1 7 9
📝 Code C++ — Backtracking 
#include <bits/stdc++.h>
using namespace std;
int board[9][9];
bool rowUsed[9][10], colUsed[9][10], boxUsed[9][10];
vector<pair<int,int>> blanks;

bool solve(int idx) {
    if (idx == (int)blanks.size()) return true;
    int r = blanks[idx].first, c = blanks[idx].second;
    int b = (r / 3) * 3 + c / 3;
    for (int v = 1; v <= 9; v++) {
        if (!rowUsed[r][v] && !colUsed[c][v] && !boxUsed[b][v]) {
            board[r][c] = v;
            rowUsed[r][v] = colUsed[c][v] = boxUsed[b][v] = true;
            if (solve(idx + 1)) return true;
            rowUsed[r][v] = colUsed[c][v] = boxUsed[b][v] = false;
        }
    }
    return false;
}

int main() {
    ifstream in("SUDOKU.INP");
    ofstream out("SUDOKU.OUT");
    for (int i = 0; i < 9; i++)
        for (int j = 0; j < 9; j++) {
            in >> board[i][j];
            if (board[i][j] == 0) blanks.push_back({i, j});
            else {
                int v = board[i][j], b = (i/3)*3+j/3;
                rowUsed[i][v] = colUsed[j][v] = boxUsed[b][v] = true;
            }
        }
    solve(0);
    for (int i = 0; i < 9; i++) {
        for (int j = 0; j < 9; j++)
            out << board[i][j] << " \n"[j == 8];
    }
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Dùng Quay lui (B1d4). Liệt kê ô trống. Với mỗi ô trống, thử giá trị 1-9, kiểm tra 3 ràng buộc (hàng, cột, ô 3×3) bằng mảng bool. Nếu hợp lệ → đánh dấu, đệ quy ô tiếp theo. Nếu không tìm được → quay lui (bỏ đánh dấu). Nhờ 3 ràng buộc, hầu hết nhánh bị cắt sớm → chạy nhanh dù 981 lý thuyết rất lớn.
Bài 7: XÂU NGOẶC HỢP LỆ
XAUNGOAC.INP / XAUNGOAC.OUT

Cho số nguyên N. Đếm số xâu ngoặc hợp lệ có độ dài 2N (chỉ dùng ký tự '(' và ')').

Dữ liệu vào (XAUNGOAC.INP):
  • Một số nguyên N (1 ≤ N ≤ 12)
Kết quả ra (XAUNGOAC.OUT):
  • Một số nguyên — số xâu ngoặc hợp lệ

XAUNGOAC.INP

3

XAUNGOAC.OUT

5
Giải thích: 5 xâu hợp lệ: ((())), (()()), (())(), ()(()), ()()().
📝 Code C++ — Quay lui (Backtracking) 
#include <bits/stdc++.h>
using namespace std;
int n, ans;

void solve(int pos, int open, int close) {
    if (pos == 2 * n) { ans++; return; }
    if (open < n)
        solve(pos + 1, open + 1, close);
    if (close < open)
        solve(pos + 1, open, close + 1);
}

int main() {
    ifstream in("XAUNGOAC.INP");
    ofstream out("XAUNGOAC.OUT");
    in >> n;
    ans = 0;
    solve(0, 0, 0);
    out << ans;
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Dùng Quay lui (B1d1). Tại mỗi vị trí, thử đặt '(' hoặc ')'. Cắt nhánh: số '(' không vượt N, số ')' không vượt số '(' đã đặt. Khi đặt đủ 2N ký tự → đếm 1 cách. N ≤ 12 → Catalan(12) = 208012, chạy rất nhanh.
Bài 8: TÔ MÀU ĐỒ THỊ
TOMAU.INP / TOMAU.OUT

Cho đồ thị vô hướng N đỉnh, M cạnh. Tìm số màu K nhỏ nhất để tô N đỉnh sao cho hai đỉnh kề nhau luôn có màu khác nhau (sắc số đồ thị).

Dữ liệu vào (TOMAU.INP):
  • Dòng 1: N, M (1 ≤ N ≤ 12; 0 ≤ M ≤ N×(N−1)/2)
  • M dòng tiếp: u, v (1 ≤ u, v ≤ N) — cạnh nối u và v
Kết quả ra (TOMAU.OUT):
  • Một số nguyên K — số màu nhỏ nhất cần dùng

TOMAU.INP

4 4
1 2
2 3
3 4
4 1

TOMAU.OUT

2
Giải thích: Đồ thị chu trình C4. Tô: đỉnh 1,3 màu 1; đỉnh 2,4 màu 2. Cần ít nhất 2 màu.
📝 Code C++ — Quay lui + cắt nhánh 
#include <bits/stdc++.h>
using namespace std;
int n, m, color[15];
bool adj[15][15];

bool canColor(int u, int c) {
    for (int v = 1; v <= n; v++)
        if (adj[u][v] && color[v] == c) return false;
    return true;
}

bool solve(int u, int k) {
    if (u > n) return true;
    for (int c = 1; c <= k; c++) {
        if (canColor(u, c)) {
            color[u] = c;
            if (solve(u + 1, k)) return true;
            color[u] = 0;
        }
    }
    return false;
}

int main() {
    ifstream in("TOMAU.INP");
    ofstream out("TOMAU.OUT");
    in >> n >> m;
    for (int i = 0; i < m; i++) {
        int u, v; in >> u >> v;
        adj[u][v] = adj[v][u] = true;
    }
    for (int k = 1; k <= n; k++)
        if (solve(1, k)) {
            out << k;
            in.close();
            out.close();

            return 0;
        }
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Dùng Quay lui (B1d1). Thử lần lượt K = 1, 2, 3,… Với mỗi K, quay lui tô từng đỉnh: thử K màu, kiểm tra xung đột với đỉnh kề đã tô. Cắt nhánh: nếu không tô được đỉnh hiện tại → quay lui. N ≤ 12, K thường rất nhỏ (≤ 4 theo định lý 4 màu cho đồ thị phẳng) → chạy nhanh.
Bài 9: PHÂN TÍCH SỐ
CHIASO.INP / CHIASO.OUT

Cho số nguyên dương N. Đếm số cách phân tích N thành tổng các số nguyên dương theo thứ tự không giảm (không tính phân tích chỉ gồm N).

Dữ liệu vào (CHIASO.INP):
  • Một số nguyên N (2 ≤ N ≤ 30)
Kết quả ra (CHIASO.OUT):
  • Dòng 1: Số cách phân tích
  • Các dòng tiếp: mỗi dòng là một cách phân tích (các số cách nhau bởi dấu cộng)

CHIASO.INP

5

CHIASO.OUT

6
1+1+1+1+1
1+1+1+2
1+1+3
1+2+2
1+4
2+3
Giải thích: 6 cách phân tích 5 thành tổng (không tính "5" đứng một mình).
📝 Code C++ — Quay lui liệt kê 
#include <bits/stdc++.h>
using namespace std;
int n, cnt;
vector<int> cur;
ofstream out("CHIASO.OUT");

void solve(int remain, int minVal) {
    if (remain == 0 && cur.size() > 1) {
        cnt++;
        for (int i = 0; i < (int)cur.size(); i++) {
            if (i) out << "+";
            out << cur[i];
        }
        out << "\n";
        return;
    }
    for (int v = minVal; v <= remain; v++) {
        cur.push_back(v);
        solve(remain - v, v);
        cur.pop_back();
    }
}

int main() {
    ifstream in("CHIASO.INP");
    in >> n;
    cnt = 0;
    solve(n, 1);
    // Ghi số cách lên đầu file: đọc lại và chèn
    out.close();
    ifstream tmp("CHIASO.OUT");
    string all((istreambuf_iterator<char>(tmp)), istreambuf_iterator<char>());
    tmp.close();
    ofstream out("CHIASO.OUT");
    out << cnt << "\n" << all;
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Dùng Quay lui liệt kê (B1d1). solve(remain, minVal): còn cần tổng remain, số tiếp theo ≥ minVal (đảm bảo không giảm). Thử v từ minVal đến remain, đệ quy với (remain − v, v). Khi remain = 0 và có ≥ 2 số → in 1 cách. Cắt nhánh tự nhiên: v ≤ remain. N ≤ 30 → partition(30) = 5604, rất nhanh.
Bài 10: NHÁNH CẬN
NHCANH.INP / NHCANH.OUT

Có N món hàng, món thứ i có trọng lượng wi và giá trị vi. Ba lô chứa tối đa W kg. Chọn các món hàng (mỗi món chọn hoặc không) sao cho tổng trọng lượng ≤ W, tổng giá trị lớn nhất.

Ràng buộc: N ≤ 30, W ≤ 1015 (N nhỏ nhưng W rất lớn → không dùng DP theo W được).

Dữ liệu vào (NHCANH.INP):
  • Dòng 1: N, W
  • N dòng tiếp: wi, vi (1 ≤ wi, vi ≤ 109)
Kết quả ra (NHCANH.OUT):
  • Một số nguyên — tổng giá trị lớn nhất

NHCANH.INP

4 10
5 10
4 8
6 12
3 7

NHCANH.OUT

20
Giải thích: Chọn món 2 (w=4, v=8) và món 3 (w=6, v=12): w=4+6=10 ≤ 10, v=8+12=20. Đây là phương án tối ưu → đáp án = 20.
📝 Code C++ — Nhánh cận (Branch and Bound) 
#include <bits/stdc++.h>
using namespace std;
int n;
long long W, best = 0;
vector<pair<long long,long long>> items; // {w, v} sắp theo v/w giảm

// Cận trên: giá trị hiện tại + fractional knapsack phần còn lại
double bound(int i, long long w, long long v) {
    double res = v;
    long long rem = W - w;
    for (int j = i; j < n && rem > 0; j++) {
        if (items[j].first <= rem) {
            rem -= items[j].first;
            res += items[j].second;
        } else {
            res += (double)items[j].second * rem / items[j].first;
            break;
        }
    }
    return res;
}

void solve(int i, long long w, long long v) {
    best = max(best, v);
    if (i == n) return;
    // Cắt nhánh: nếu cận trên ≤ best → bỏ
    if (bound(i, w, v) <= best) return;
    // Thử chọn
    if (w + items[i].first <= W)
        solve(i + 1, w + items[i].first, v + items[i].second);
    // Thử không chọn
    solve(i + 1, w, v);
}

int main() {
    ifstream in("NHCANH.INP");
    ofstream out("NHCANH.OUT");
    in >> n >> W;
    items.resize(n);
    for (int i = 0; i < n; i++) in >> items[i].first >> items[i].second;
    // Sắp theo đơn giá giảm
    sort(items.begin(), items.end(), [](const pair<long long,long long>& a, const pair<long long,long long>& b) {
        return (double)a.second / a.first > (double)b.second / b.first;
    });
    solve(0, 0, 0);
    out << best;
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Dùng Nhánh cận (B1d4). N ≤ 30 nhưng W rất lớn (1015) → không dùng DP theo W. Sắp theo đơn giá giảm. Tại mỗi bước: tính cận trên = giá trị hiện tại + Fractional Knapsack phần còn lại (tham lam lấy phần). Nếu cận trên ≤ best đã tìm → cắt nhánh (không cần duyệt tiếp).
