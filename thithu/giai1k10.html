Bài 1: Ghép cặp thi đấu (GHEPCAP)
📝 Code C++ — Sort + Greedy O(N log N) 
// Bai 1: GHEPCAP — Sort + Greedy O(N log N)
#include <bits/stdc++.h>
using namespace std;

int main() {
    ifstream in("GHEPCAP.INP");
    ofstream out("GHEPCAP.OUT");

    int n;
    in >> n;
    vector<int> a(2 * n);
    for (int i = 0; i < 2 * n; i++) in >> a[i];

    sort(a.begin(), a.end());

    long long ans = 0;
    for (int i = 0; i < 2 * n; i += 2)
        ans += a[i + 1] - a[i];

    out << ans;

    in.close();
    out.close();
    return 0;
}
💡 Phân tích:
Subtask 1 (N ≤ 8): Duyệt tất cả cách ghép cặp. Với 2N=16 người, số cách ghép = (2N−1)!! = 15×13×…×1 ≈ 2 triệu. Dùng đệ quy: mỗi bước chọn người đầu tiên chưa ghép, duyệt tất cả đối thủ còn lại.
Subtask 2 (Full): Nhận xét quan trọng: sort tăng dần rồi ghép cặp liền kề luôn cho kết quả tối ưu.

Tại sao đúng? Giả sử có 4 người a ≤ b ≤ c ≤ d. So sánh 2 cách ghép:
  • Ghép liền kề: (a,b) + (c,d) = (b−a) + (d−c)
  • Ghép chéo: (a,c) + (b,d) = (c−a) + (d−b) = d + c − a − b ≥ (b−a) + (d−c)
  → Ghép liền kề luôn ≤ ghép chéo. Quy nạp cho N cặp → sort + ghép liền kề tối ưu.
Tổng: O(N log N) cho sort, O(N) cho tính tổng.
Bài 2: Doanh thu mùa hè (TONGMAX)
📝 Code C++ — Kadane O(N) 
// Bai 2: TONGMAX — Kadane O(N)
#include <bits/stdc++.h>
using namespace std;

int main() {
    ifstream in("TONGMAX.INP");
    ofstream out("TONGMAX.OUT");

    int n;
    in >> n;

    long long cur = 0, ans;
    for (int i = 0; i < n; i++) {
        long long x;
        in >> x;
        cur += x;
        if (i == 0) ans = cur;
        else ans = max(ans, cur);
        if (cur < 0) cur = 0;
    }

    out << ans;

    in.close();
    out.close();
    return 0;
}
💡 Phân tích:
Subtask 1 (N ≤ 1000): Duyệt tất cả cặp (l, r): tổng = prefix[r] − prefix[l−1]. Hai vòng for O(N²) ≈ 10⁶.
Subtask 2 (Full): Thuật toán Kadane — DP 1 biến:

Ý tưởng: Duy trì biến cur = tổng đoạn con kết thúc tại vị trí hiện tại. Tại mỗi bước:
  1. Cộng a[i] vào cur
  2. Cập nhật ans = max(ans, cur)
  3. Nếu cur < 0 → reset cur = 0 (bỏ đoạn cũ, bắt đầu đoạn mới từ vị trí sau)

Tại sao đúng? Nếu tổng tích lũy âm thì không bao giờ có lợi khi giữ lại, vì bắt đầu đoạn mới từ vị trí kế tiếp luôn tốt hơn.
Lưu ý: Khi dãy toàn âm, ans sẽ là phần tử lớn nhất (ít âm nhất). Vì ta cập nhật ans TRƯỚC khi reset cur.
Tổng: O(N) — chỉ duyệt 1 lần.
Bài 3: Thu hoạch vàng (THUVANG)
📝 Code C++ — DP 1D O(N) 
// Bai 3: THUVANG — DP 1D O(N)
#include <bits/stdc++.h>
using namespace std;

int main() {
    ifstream in("THUVANG.INP");
    ofstream out("THUVANG.OUT");

    int n;
    in >> n;
    vector<long long> a(n + 1);
    for (int i = 1; i <= n; i++) in >> a[i];

    vector<long long> dp(n + 1, 0);
    dp[1] = a[1];
    if (n >= 2) dp[2] = a[1] + a[2];
    for (int i = 3; i <= n; i++)
        dp[i] = a[i] + max(dp[i - 1], dp[i - 2]);

    out << dp[n];

    in.close();
    out.close();
    return 0;
}
💡 Phân tích:
Subtask 1 (N ≤ 20): Đệ quy thử tất cả đường đi. Tại mỗi ô, có 2 lựa chọn (nhảy 1 hoặc 2) → tối đa 2²⁰ ≈ 10⁶ trạng thái.
Subtask 2 (Full): Quy hoạch động 1 chiều:

Định nghĩa: dp[i] = tổng vàng lớn nhất khi đi từ ô 1 đến ô i.
Cơ sở:
  • dp[1] = a[1] (bắt buộc xuất phát tại ô 1)
  • dp[2] = a[1] + a[2] (chỉ có một cách: 1→2)
Chuyển trạng thái (i ≥ 3):
  • dp[i] = a[i] + max(dp[i−1], dp[i−2])
  • Ý nghĩa: đến ô i phải đi qua ô i (thu a[i]), và trước đó ta ở ô i−1 hoặc ô i−2 (chọn cái nào tốt hơn).

Kết quả: dp[N].
Tổng: O(N) — duyệt 1 lần từ 3 đến N.
