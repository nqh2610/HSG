Bài 1: Phố ẩm thực (AMTHUC)
📝 Code C++ — Prefix Sum + Map O(N log N) 
// Bai 1: AMTHUC — Prefix Sum + Map O(N log N)
#include <bits/stdc++.h>
using namespace std;

int main() {
    ifstream in("AMTHUC.INP");
    ofstream out("AMTHUC.OUT");

    int n;
    in >> n;

    map<long long, long long> cnt;
    long long prefix = 0, ans = 0;
    cnt[0] = 1;

    for (int i = 0; i < n; i++) {
        long long x;
        in >> x;
        prefix += x;
        ans += cnt[prefix];
        cnt[prefix]++;
    }

    out << ans;

    in.close();
    out.close();
    return 0;
}
💡 Phân tích:
Subtask 1 (N ≤ 1000): Duyệt mọi cặp (l, r), tính tổng đoạn, đếm đoạn có tổng = 0. Dùng prefix sum tính tổng O(1). Tổng O(N²) ≈ 10⁶.
Subtask 2 (Full): Prefix Sum + Đếm tần số:

Nhận xét quan trọng: Gọi prefix[i] = a[1] + a[2] + … + a[i], prefix[0] = 0.
Tổng đoạn [l, r] = prefix[r] − prefix[l−1].
Đoạn có tổng = 0 ⟺ prefix[r] = prefix[l−1].

Bài toán quy về: Đếm số cặp (i, j) với i < j mà prefix[i] = prefix[j].

Cách làm: Duyệt i từ 0 đến N, dùng map đếm số lần mỗi giá trị prefix đã xuất hiện. Khi gặp prefix[i], cộng thêm số lần giá trị đó đã xuất hiện trước đó vào đáp án.

Lưu ý: Dùng long long cho prefix vì tổng có thể đạt 10⁵ × 10⁹ = 10¹⁴. Đáp án cũng cần long long (tối đa C(N+1, 2) ≈ 5 × 10⁹).
Độ phức tạp: O(N log N) thời gian (do map), O(N) bộ nhớ.
Bài 2: Đấu giao hữu (DOITHI)
📝 Code C++ — Sort + Two Pointers O(N log N) 
// Bai 2: DOITHI — Sort + Two Pointers O(N log N)
#include <bits/stdc++.h>
using namespace std;

int main() {
    ifstream in("DOITHI.INP");
    ofstream out("DOITHI.OUT");

    int n;
    long long d;
    in >> n >> d;

    vector<long long> a(n);
    for (int i = 0; i < n; i++) in >> a[i];

    sort(a.begin(), a.end());

    long long ans = 0;
    int left = 0;
    for (int right = 0; right < n; right++) {
        while (a[right] - a[left] > d) left++;
        ans += right - left;
    }

    out << ans;

    in.close();
    out.close();
    return 0;
}
💡 Phân tích:
Subtask 1 (N ≤ 1000): Duyệt mọi cặp (i, j), kiểm tra |a[i] − a[j]| ≤ D. Tổng O(N²) ≈ 10⁶.
Subtask 2 (Full): Sắp xếp + Hai con trỏ (Two Pointers):

Bước 1 — Sắp xếp: Sau khi sắp xếp tăng dần, |a[i] − a[j]| = a[j] − a[i] với j > i.

Bước 2 — Hai con trỏ: Duyệt right từ trái sang phải. Với mỗi right, tìm vị trí left nhỏ nhất sao cho a[right] − a[left] ≤ D.
→ Tất cả vị trí từ left đến right−1 đều ghép được với right.
→ Số cặp mới = right − left.

Tại sao hiệu quả? Con trỏ left chỉ tăng (không bao giờ giảm) vì khi right tăng, a[right] tăng → left chỉ có thể tăng hoặc giữ nguyên. Tổng số bước di chuyển của left ≤ N.

Lưu ý: Đáp án cần long long (tối đa C(N, 2) ≈ 5 × 10⁹).
Độ phức tạp: O(N log N) do sắp xếp, phần two pointers O(N).
Bài 3: Bộ sưu tập (BSUTAP)
📝 Code C++ — Sliding Window O(N) 
// Bai 3: BSUTAP — Sliding Window O(N)
#include <bits/stdc++.h>
using namespace std;

int main() {
    ifstream in("BSUTAP.INP");
    ofstream out("BSUTAP.OUT");

    int n, k;
    in >> n >> k;

    vector<int> a(n);
    for (int i = 0; i < n; i++) in >> a[i];

    int freq[100001] = {};
    int distinct = 0, ans = 0, left = 0;

    for (int right = 0; right < n; right++) {
        if (freq[a[right]] == 0) distinct++;
        freq[a[right]]++;

        while (distinct > k) {
            freq[a[left]]--;
            if (freq[a[left]] == 0) distinct--;
            left++;
        }

        if (right - left + 1 > ans) ans = right - left + 1;
    }

    out << ans;

    in.close();
    out.close();
    return 0;
}
💡 Phân tích:
Subtask 1 (N ≤ 1000): Duyệt mọi đoạn [l, r], đếm số loại khác nhau bằng mảng đánh dấu. Tổng O(N²) hoặc O(N² × max_a).
Subtask 2 (Full): Kĩ thuật cửa sổ trượt (Sliding Window):

Ý tưởng: Duy trì một "cửa sổ" [left, right] luôn thỏa mãn: số loại khác nhau trong cửa sổ ≤ K.

Cách làm:
  • Mở rộng cửa sổ bằng cách tăng right: thêm a[right] vào cửa sổ, cập nhật tần số.
  • Nếu số loại distinct > K: thu hẹp cửa sổ bằng cách tăng left, giảm tần số. Khi tần số về 0 thì distinct giảm 1.
  • Cập nhật ans = max(ans, right − left + 1).

Tại sao đúng? Cửa sổ [left, right] luôn chứa ≤ K loại. Khi right tăng mà vi phạm, ta chỉ cần tăng left vừa đủ để khôi phục điều kiện. Mọi đoạn hợp lệ đều được xét qua.

Mảng tần số: Dùng freq[100001] (vì a[i] ≤ 10⁵) thay cho map để truy cập O(1).
Độ phức tạp: O(N) thời gian (mỗi phần tử vào/ra cửa sổ tối đa 1 lần), O(max_a) bộ nhớ.
