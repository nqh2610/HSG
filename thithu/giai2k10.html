Bài 1: Đếm số may mắn (DEMSO)
📝 Code C++ — Sàng Eratosthenes O(R log log R) 
// Bai 1: DEMSO — Sieve of Eratosthenes O(R log log R)
#include <bits/stdc++.h>
using namespace std;

int main() {
    ifstream in("DEMSO.INP");
    ofstream out("DEMSO.OUT");

    int l, r;
    in >> l >> r;

    vector<bool> sieve(r + 1, true);
    sieve[0] = sieve[1] = false;
    for (int i = 2; (long long)i * i <= r; i++)
        if (sieve[i])
            for (int j = i * i; j <= r; j += i)
                sieve[j] = false;

    int cnt = 0;
    for (int i = l; i <= r; i++)
        if (sieve[i]) cnt++;

    out << cnt;

    in.close();
    out.close();
    return 0;
}
💡 Phân tích:
Subtask 1 (R ≤ 1000): Kiểm tra từng số x trong [L, R]: duyệt ước từ 2 đến √x, nếu không chia hết → nguyên tố. Tổng O((R−L) × √R) ≈ 30 000.
Subtask 2 (Full): Sàng Eratosthenes:

Ý tưởng: Tạo mảng boolean sieve[0..R], ban đầu tất cả = true (giả sử nguyên tố).
  1. Đánh dấu sieve[0] = sieve[1] = false (0 và 1 không phải nguyên tố)
  2. Duyệt i từ 2 đến √R: nếu sieve[i] = true, thì đánh dấu tất cả bội của i (i², i²+i, i²+2i, …) = false
  3. Sau sàng, đếm các sieve[i] = true trong đoạn [L, R]

Tại sao bắt đầu từ i²? Vì các bội nhỏ hơn i² (như 2i, 3i, …) đã bị đánh dấu bởi các số nguyên tố nhỏ hơn i rồi.
Tổng: O(R log log R) ≈ 4 × 10⁶ — rất nhanh cho R ≤ 10⁶.
Bài 2: Chọn nhóm bạn (DOANDEP)
📝 Code C++ — Two Pointers O(N) 
// Bai 2: DOANDEP — Two Pointers O(N)
#include <bits/stdc++.h>
using namespace std;

int main() {
    ifstream in("DOANDEP.INP");
    ofstream out("DOANDEP.OUT");

    int n;
    long long k;
    in >> n >> k;

    vector<long long> a(n);
    for (int i = 0; i < n; i++) in >> a[i];

    long long ans = 0, sum = 0;
    int l = 0;
    for (int r = 0; r < n; r++) {
        sum += a[r];
        while (sum > k) {
            sum -= a[l];
            l++;
        }
        ans += r - l + 1;
    }

    out << ans;

    in.close();
    out.close();
    return 0;
}
💡 Phân tích:
Subtask 1 (N ≤ 1000): Duyệt tất cả cặp (l, r): tính tổng bằng prefix sum, đếm nếu tổng ≤ K. Tổng O(N²) ≈ 10⁶.
Subtask 2 (Full): Kỹ thuật Hai con trỏ (Two Pointers):

Nhận xét: Vì a[i] > 0 nên khi mở rộng đoạn (tăng r) → tổng tăng, thu hẹp (tăng l) → tổng giảm. Tính chất đơn điệu này cho phép dùng 2 con trỏ.

Thuật toán: Duy trì cửa sổ [l, r] với tổng sum ≤ K:
  1. Tăng r, cộng a[r] vào sum
  2. Nếu sum > K, tăng l (trừ a[l]) cho đến khi sum ≤ K
  3. Lúc này tất cả đoạn kết thúc tại r có l' ∈ [l, r] đều thỏa mãn → đếm (r − l + 1) đoạn

Tại sao đúng? Vì dãy toàn dương, nếu [l, r] thỏa mãn thì mọi đoạn con [l', r] với l' ≥ l cũng thỏa.
Lưu ý: l chỉ tăng chứ không giảm → mỗi phần tử bị thêm vào 1 lần, bị bỏ ra tối đa 1 lần → tổng O(N).
Bài 3: Sửa biểu thức (SUANGOAC)
📝 Code C++ — Greedy O(N) 
// Bai 3: SUANGOAC — Greedy O(N)
#include <bits/stdc++.h>
using namespace std;

int main() {
    ifstream in("SUANGOAC.INP");
    ofstream out("SUANGOAC.OUT");

    string s;
    in >> s;

    int open = 0, add = 0;
    for (char c : s) {
        if (c == '(') open++;
        else {
            if (open > 0) open--;
            else add++;
        }
    }

    out << open + add;

    in.close();
    out.close();
    return 0;
}
💡 Phân tích:
Subtask 1 (|S| ≤ 100): Mô phỏng: lặp lại quá trình "tìm cặp () hợp lệ → xóa" cho đến khi không còn cặp nào. Số kí tự còn lại chính là số cần thêm.
Subtask 2 (Full): Tham lam (Greedy) với 2 biến đếm:

Ý tưởng: Duy trì biến open = số dấu '(' chưa được ghép.
  • Gặp '(' → open++ (thêm 1 ngoặc mở chờ ghép)
  • Gặp ')' → nếu có '(' chờ (open > 0) thì ghép: open--. Nếu không có '(' nào chờ → ')' này thừa, cần thêm 1 dấu '(' → add++.

Sau khi duyệt xong:
  • open = số '(' chưa được ghép → cần thêm open dấu ')'
  • add = số ')' thừa → cần thêm add dấu '('
  • Kết quả = open + add

Ví dụ ")()(": ')' → add=1, '(' → open=1, ')' → ghép open=0, '(' → open=1. Kết quả = 1+1 = 2 ✅
Tổng: O(N) — duyệt 1 lần.
