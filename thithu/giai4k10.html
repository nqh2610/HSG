Bài 1: Tưới cây (TUOICAY)
📝 Code C++ — Difference Array O(N+M) 
// Bai 1: TUOICAY — Difference Array O(N+M)
#include <bits/stdc++.h>
using namespace std;

int main() {
    ifstream in("TUOICAY.INP");
    ofstream out("TUOICAY.OUT");

    int n, m;
    in >> n >> m;

    vector<long long> diff(n + 2, 0);
    for (int i = 0; i < m; i++) {
        int l, r;
        long long v;
        in >> l >> r >> v;
        diff[l] += v;
        diff[r + 1] -= v;
    }

    long long mx = 0, cur = 0;
    for (int i = 1; i <= n; i++) {
        cur += diff[i];
        mx = max(mx, cur);
    }

    out << mx;

    in.close();
    out.close();
    return 0;
}
💡 Phân tích:
Subtask 1 (N, M ≤ 1000): Mỗi lần tưới, duyệt từ l đến r, cộng v cho mỗi cây. Tổng O(N × M) ≈ 10⁶.
Subtask 2 (Full): Mảng hiệu (Difference Array):

Ý tưởng: Thay vì cộng v cho mỗi phần tử trong [l, r] (tốn O(r−l+1)), ta chỉ cần:
  • diff[l] += v (đánh dấu "bắt đầu cộng từ đây")
  • diff[r+1] −= v (đánh dấu "hết cộng sau đây")
→ Mỗi thao tác chỉ tốn O(1).

Khôi phục giá trị thực: Sau M thao tác, duyệt prefix sum trên mảng diff:
  cur += diff[i] → cur chính là lượng nước cây i nhận được.

Tại sao đúng? Khi cộng diff[l] = +v, tất cả prefix sum từ vị trí l trở đi đều tăng v. Khi trừ diff[r+1] = −v, từ vị trí r+1 trở đi giảm v, triệt tiêu. Kết quả: chỉ [l, r] tăng v.

Lưu ý: Dùng long long vì tổng có thể đạt M × v = 10⁵ × 10⁹ = 10¹⁴.
Độ phức tạp: O(N + M) thời gian, O(N) bộ nhớ.
Bài 2: Đôi bạn (DOIBAN)
📝 Code C++ — Đếm tần số O(N) 
// Bai 2: DOIBAN — Counting O(N)
#include <bits/stdc++.h>
using namespace std;

int main() {
    ifstream in("DOIBAN.INP");
    ofstream out("DOIBAN.OUT");

    int n;
    in >> n;

    int freq[100001] = {};
    for (int i = 0; i < n; i++) {
        int x;
        in >> x;
        freq[x]++;
    }

    long long ans = 0;
    for (int c = 1; c <= 100000; c++) {
        if (freq[c] >= 2)
            ans += (long long)freq[c] * (freq[c] - 1) / 2;
    }

    out << ans;

    in.close();
    out.close();
    return 0;
}
💡 Phân tích:
Subtask 1 (N ≤ 1000): Duyệt mọi cặp (i, j) với i < j, kiểm tra a[i] = a[j]. Tổng O(N²) ≈ 10⁶.
Subtask 2 (Full): Đếm tần số (Counting):

Nhận xét: Nếu sở thích c xuất hiện freq[c] lần, thì số đôi bạn cùng sở thích c = C(freq[c], 2) = freq[c] × (freq[c] − 1) / 2.

Cách làm:
  • Bước 1: Đếm tần số mỗi giá trị bằng mảng freq[100001].
  • Bước 2: Duyệt qua tất cả giá trị c, cộng C(freq[c], 2) vào đáp án.

Tại sao C(f, 2)? Từ f người cùng sở thích, chọn 2 người bất kỳ = f! / (2! × (f−2)!) = f × (f−1) / 2.

Lưu ý: Đáp án cần long long — trường hợp xấu nhất: tất cả N = 10⁵ người cùng sở thích → C(10⁵, 2) ≈ 5 × 10⁹.
Độ phức tạp: O(N + max_a) thời gian, O(max_a) bộ nhớ.
Bài 3: Trận đấu (TRANDAU)
📝 Code C++ — Kadane's Algorithm O(N) 
// Bai 3: TRANDAU — Kadane O(N)
#include <bits/stdc++.h>
using namespace std;

int main() {
    ifstream in("TRANDAU.INP");
    ofstream out("TRANDAU.OUT");

    int n;
    in >> n;
    long long x, maxEnd, maxSoFar;
    in >> x;
    maxEnd = maxSoFar = x;

    for (int i = 1; i < n; i++) {
        in >> x;
        maxEnd = max(x, maxEnd + x);
        maxSoFar = max(maxSoFar, maxEnd);
    }

    out << maxSoFar;

    in.close();
    out.close();
    return 0;
}
💡 Phân tích:
Subtask 1 (N ≤ 1000): Duyệt mọi cặp (l, r), tính tổng đoạn, cập nhật max. Dùng prefix sum có thể tính tổng O(1). Tổng O(N²) ≈ 10⁶.
Subtask 2 (Full): Thuật toán Kadane O(N):

Ý tưởng: Duy trì 2 biến:
  • maxEnd = tổng lớn nhất của đoạn con kết thúc tại vị trí hiện tại
  • maxSoFar = tổng lớn nhất toàn bộ (đáp án)

Chuyển trạng thái: Tại vị trí i, đoạn con kết thúc tại i hoặc:
  • Nối tiếp đoạn trước: maxEnd + a[i]
  • Bắt đầu đoạn mới từ a[i]: a[i]
→ maxEnd = max(a[i], maxEnd + a[i])

Tại sao đúng? Nếu maxEnd < 0, đoạn cũ chỉ "kéo giảm" tổng → bắt đầu lại từ a[i] tốt hơn. Ngược lại, maxEnd ≥ 0 → nối thêm a[i] chỉ có lợi hoặc không thay đổi.

Lưu ý: Dùng long long vì tổng có thể đạt 10⁵ × 10⁹ = 10¹⁴. Khi tất cả a[i] âm, đáp án là phần tử lớn nhất (ít âm nhất).
Độ phức tạp: O(N) thời gian, O(1) bộ nhớ phụ.
