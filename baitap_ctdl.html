Bài Tập Cấu Trúc Dữ Liệu Nâng Cao

Bộ đề rèn luyện DSU, BIT/Fenwick Tree, Segment Tree dành cho HSG Tin học — tối ưu truy vấn tránh TLE

Bài 1: NHÓM BẠN
NHOMBAN.INP / NHOMBAN.OUT

Có N học sinh đánh số 1 đến N. Ban đầu mỗi em là 1 nhóm riêng. Có Q thao tác:

• 1 u v: Hợp nhất nhóm chứa u và nhóm chứa v.
• 2 u v: Hỏi u và v có cùng nhóm không?

Dữ liệu vào (NHOMBAN.INP):
  • Dòng 1: N, Q (1 ≤ N, Q ≤ 106)
  • Q dòng tiếp: mỗi dòng 1 thao tác
Kết quả ra (NHOMBAN.OUT):
  • Với mỗi thao tác loại 2, in "YES" hoặc "NO"

NHOMBAN.INP

5 5
1 1 2
1 3 4
2 1 2
2 1 3
1 2 3

NHOMBAN.OUT

YES
NO
📝 Code C++ — DSU (Disjoint Set Union) 
#include <bits/stdc++.h>
using namespace std;
const int MAXN = 1e6 + 5;
int par[MAXN], rnk[MAXN];

int find(int x) {
    while (par[x] != x) x = par[x] = par[par[x]]; // path compression
    return x;
}

void unite(int a, int b) {
    a = find(a); b = find(b);
    if (a == b) return;
    if (rnk[a] < rnk[b]) swap(a, b);
    par[b] = a;
    if (rnk[a] == rnk[b]) rnk[a]++;
}

int main() {
    ifstream in("NHOMBAN.INP");
    ofstream out("NHOMBAN.OUT");
    int n, q;
    in >> n >> q;
    for (int i = 1; i <= n; i++) { par[i] = i; rnk[i] = 0; }
    while (q--) {
        int t, u, v;
        in >> t >> u >> v;
        if (t == 1) unite(u, v);
        else out << (find(u) == find(v) ? "YES" : "NO") << "\n";
    }
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Dùng DSU — Disjoint Set Union (C4). Mỗi nhóm là 1 cây, đại diện = gốc. find(x): tìm gốc của x (path compression = nén đường đi: trỏ thẳng ông nội). unite(a,b): nối 2 cây theo rank (cây thấp nối vào cây cao). Cả find và unite gần O(1) mỗi thao tác (chính xác O(α(N)) ≈ hằng số). Nếu dùng mảng nhóm + duyệt đổi nhóm → O(N) mỗi thao tác → TLE.
Bài 2: ĐẾM SỐ NHỎ HƠN
DEMLON.INP / DEMLON.OUT

Cho mảng N phần tử. Với mỗi phần tử ai, đếm số phần tử đứng trước nó (j < i) mà aj < ai.

Dữ liệu vào (DEMLON.INP):
  • Dòng 1: N (1 ≤ N ≤ 2×105)
  • Dòng 2: N số nguyên ai (1 ≤ ai ≤ 109)
Kết quả ra (DEMLON.OUT):
  • Một dòng N số nguyên, số thứ i là số phần tử đứng trước ai nhỏ hơn ai

DEMLON.INP

5
3 1 4 1 5

DEMLON.OUT

0 0 2 0 4
Giải thích: Trước 5 có {3,1,4,1}, tất cả < 5 → 4.
📝 Code C++ — BIT (Fenwick Tree) + nén tọa độ 
#include <bits/stdc++.h>
using namespace std;
const int MAXN = 2e5 + 5;
int bit[MAXN], n;

void update(int i) {
    for (; i <= n; i += i & (-i)) bit[i]++;
}

int query(int i) {
    int s = 0;
    for (; i > 0; i -= i & (-i)) s += bit[i];
    return s;
}

int main() {
    ifstream in("DEMLON.INP");
    ofstream out("DEMLON.OUT");
    in >> n;
    vector<int> a(n);
    for (int i = 0; i < n; i++) in >> a[i];
    // Nén tọa độ
    vector<int> sorted_a = a;
    sort(sorted_a.begin(), sorted_a.end());
    sorted_a.erase(unique(sorted_a.begin(), sorted_a.end()), sorted_a.end());
    for (int i = 0; i < n; i++)
        a[i] = lower_bound(sorted_a.begin(), sorted_a.end(), a[i]) - sorted_a.begin() + 1;
    // Duyệt từ trái sang phải
    for (int i = 0; i < n; i++) {
        out << query(a[i] - 1) << " \n"[i == n - 1];
        update(a[i]);
    }
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Dùng BIT — Fenwick Tree (C5) + nén tọa độ. Duyệt brute → O(N2) TLE. Nén tọa độ: ánh xạ giá trị về 1..N. BIT lưu tần suất xuất hiện. Khi xử lý a[i]: query(a[i]−1) = đếm số phần tử đã thêm có giá trị < a[i]. Rồi update(a[i]) = thêm a[i] vào BIT. Mỗi phép toán O(log N). Tổng O(N log N).
Bài 3: CẬP NHẬT ĐOẠN + TRUY VẤN ĐIỂM
CAPNHAT.INP / CAPNHAT.OUT

Cho mảng N phần tử, ban đầu toàn 0. Có Q thao tác:

• 1 L R V: Cộng V vào mọi phần tử aL, aL+1, ..., aR.
• 2 X: In giá trị aX hiện tại.

Dữ liệu vào (CAPNHAT.INP):
  • Dòng 1: N, Q (1 ≤ N, Q ≤ 5×105)
  • Q dòng tiếp: mỗi dòng 1 thao tác
Kết quả ra (CAPNHAT.OUT):
  • Với mỗi thao tác loại 2, in giá trị aX

CAPNHAT.INP

5 4
1 1 3 2
1 2 5 3
2 3
2 4

CAPNHAT.OUT

5
3
Giải thích: Sau thao tác 1: [2,2,2,0,0]. Sau thao tác 2: [2,5,5,3,3]. a[3]=5, a[4]=3.
📝 Code C++ — BIT cập nhật đoạn 
#include <bits/stdc++.h>
using namespace std;
const int MAXN = 5e5 + 5;
long long bit[MAXN];
int n;

void update(int i, long long v) {
    for (; i <= n; i += i & (-i)) bit[i] += v;
}

long long query(int i) {
    long long s = 0;
    for (; i > 0; i -= i & (-i)) s += bit[i];
    return s;
}

int main() {
    ifstream in("CAPNHAT.INP");
    ofstream out("CAPNHAT.OUT");
    int q;
    in >> n >> q;
    while (q--) {
        int t;
        in >> t;
        if (t == 1) {
            int l, r; long long v;
            in >> l >> r >> v;
            update(l, v);
            update(r + 1, -v);
        } else {
            int x;
            in >> x;
            out << query(x) << "\n";
        }
    }
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Dùng BIT (C5) trên mảng hiệu (difference array). Cập nhật đoạn [L, R] cộng V = cập nhật 2 điểm trên mảng hiệu: update(L, +V), update(R+1, −V). Truy vấn giá trị tại X = tổng tiền tố mảng hiệu = query(X). Cập nhật O(log N), truy vấn O(log N). Nếu cập nhật duyệt thô → O(N) mỗi thao tác → O(NQ) TLE.
Bài 4: TRUY VẤN MAX ĐOẠN
MAXDOAN.INP / MAXDOAN.OUT

Cho mảng N phần tử. Có Q thao tác:

• 1 P V: Gán aP = V.
• 2 L R: In max(aL, aL+1, ..., aR).

Dữ liệu vào (MAXDOAN.INP):
  • Dòng 1: N, Q (1 ≤ N, Q ≤ 2×105)
  • Dòng 2: N số nguyên ai (|ai| ≤ 109)
  • Q dòng tiếp: mỗi dòng 1 thao tác
Kết quả ra (MAXDOAN.OUT):
  • Với mỗi thao tác loại 2, in giá trị max đoạn [L, R]

MAXDOAN.INP

5 3
1 5 3 4 2
2 2 4
1 3 6
2 1 5

MAXDOAN.OUT

5
6
📝 Code C++ — Segment Tree 
#include <bits/stdc++.h>
using namespace std;
const int MAXN = 2e5 + 5;
int n, q, a[MAXN];
int tree[4 * MAXN];

void build(int id, int l, int r) {
    if (l == r) { tree[id] = a[l]; return; }
    int mid = (l + r) / 2;
    build(id * 2, l, mid);
    build(id * 2 + 1, mid + 1, r);
    tree[id] = max(tree[id * 2], tree[id * 2 + 1]);
}

void update(int id, int l, int r, int pos, int val) {
    if (l == r) { tree[id] = val; return; }
    int mid = (l + r) / 2;
    if (pos <= mid) update(id * 2, l, mid, pos, val);
    else update(id * 2 + 1, mid + 1, r, pos, val);
    tree[id] = max(tree[id * 2], tree[id * 2 + 1]);
}

int query(int id, int l, int r, int ql, int qr) {
    if (ql > r || qr < l) return INT_MIN;
    if (ql <= l && r <= qr) return tree[id];
    int mid = (l + r) / 2;
    return max(query(id * 2, l, mid, ql, qr),
               query(id * 2 + 1, mid + 1, r, ql, qr));
}

int main() {
    ifstream in("MAXDOAN.INP");
    ofstream out("MAXDOAN.OUT");
    in >> n >> q;
    for (int i = 1; i <= n; i++) in >> a[i];
    build(1, 1, n);
    while (q--) {
        int t;
        in >> t;
        if (t == 1) {
            int p, v;
            in >> p >> v;
            update(1, 1, n, p, v);
        } else {
            int l, r;
            in >> l >> r;
            out << query(1, 1, n, l, r) << "\n";
        }
    }
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Dùng Segment Tree (C7). Cây phân đoạn: mỗi nút quản lý 1 đoạn [l, r], lưu max. build O(N), update O(log N), query O(log N). Sparse Table chỉ hỗ trợ truy vấn tĩnh (không cập nhật). Segment Tree hỗ trợ cả cập nhật lẫn truy vấn. Duyệt thô: mỗi truy vấn O(N) → O(NQ) ≈ 4×1010 → TLE. Segment Tree: O((N+Q) log N).
Bài 5: CỘNG ĐOẠN + TỔNG ĐOẠN
CONGDOAN.INP / CONGDOAN.OUT

Cho mảng N phần tử. Có Q thao tác:

• 1 L R V: Cộng V vào mọi phần tử aL..aR.
• 2 L R: In tổng aL + aL+1 + ... + aR.

Dữ liệu vào (CONGDOAN.INP):
  • Dòng 1: N, Q (1 ≤ N, Q ≤ 2×105)
  • Dòng 2: N số nguyên ai (|ai| ≤ 109)
  • Q dòng tiếp: mỗi dòng 1 thao tác
Kết quả ra (CONGDOAN.OUT):
  • Với mỗi thao tác loại 2, in tổng đoạn

CONGDOAN.INP

5 3
1 2 3 4 5
1 2 4 10
2 1 5
2 3 4

CONGDOAN.OUT

45
27
Giải thích: Sau cộng: [1, 12, 13, 14, 5]. Tổng [1,5] = 45. Tổng [3,4] = 13+14 = 27.
📝 Code C++ — Lazy Segment Tree 
#include <bits/stdc++.h>
using namespace std;
const int MAXN = 2e5 + 5;
long long tree[4 * MAXN], lazy[4 * MAXN];
int a[MAXN], n, q;

void build(int id, int l, int r) {
    lazy[id] = 0;
    if (l == r) { tree[id] = a[l]; return; }
    int mid = (l + r) / 2;
    build(id * 2, l, mid);
    build(id * 2 + 1, mid + 1, r);
    tree[id] = tree[id * 2] + tree[id * 2 + 1];
}

void push(int id, int l, int r) {
    if (lazy[id] != 0) {
        int mid = (l + r) / 2;
        tree[id * 2] += lazy[id] * (mid - l + 1);
        tree[id * 2 + 1] += lazy[id] * (r - mid);
        lazy[id * 2] += lazy[id];
        lazy[id * 2 + 1] += lazy[id];
        lazy[id] = 0;
    }
}

void update(int id, int l, int r, int ql, int qr, long long v) {
    if (ql > r || qr < l) return;
    if (ql <= l && r <= qr) {
        tree[id] += v * (r - l + 1);
        lazy[id] += v;
        return;
    }
    push(id, l, r);
    int mid = (l + r) / 2;
    update(id * 2, l, mid, ql, qr, v);
    update(id * 2 + 1, mid + 1, r, ql, qr, v);
    tree[id] = tree[id * 2] + tree[id * 2 + 1];
}

long long query(int id, int l, int r, int ql, int qr) {
    if (ql > r || qr < l) return 0;
    if (ql <= l && r <= qr) return tree[id];
    push(id, l, r);
    int mid = (l + r) / 2;
    return query(id * 2, l, mid, ql, qr) + query(id * 2 + 1, mid + 1, r, ql, qr);
}

int main() {
    ifstream in("CONGDOAN.INP");
    ofstream out("CONGDOAN.OUT");
    in >> n >> q;
    for (int i = 1; i <= n; i++) in >> a[i];
    build(1, 1, n);
    while (q--) {
        int t;
        in >> t;
        if (t == 1) {
            int l, r; long long v;
            in >> l >> r >> v;
            update(1, 1, n, l, r, v);
        } else {
            int l, r;
            in >> l >> r;
            out << query(1, 1, n, l, r) << "\n";
        }
    }
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Dùng Lazy Segment Tree (C8). Segment Tree thường: cập nhật 1 điểm O(log N). Cập nhật đoạn [L,R] cộng V: nếu cập nhật từng điểm → O(N log N) mỗi thao tác → TLE. Lazy Propagation: "lười" cập nhật — chỉ đánh dấu lazy[id] += V cho nút bao trùm. Khi cần đi sâu (truy vấn hoặc cập nhật con) → push lazy xuống con. Cả update và query đều O(log N).
Bài 6: ĐẾM NGHỊCH THẾ
NGHICHDAO.INP / NGHICHDAO.OUT

Cho hoán vị P gồm N phần tử. Đếm số cặp nghịch thế (i, j) với i < j và Pi > Pj.

Dữ liệu vào (NGHICHDAO.INP):
  • Dòng 1: N (1 ≤ N ≤ 5×105)
  • Dòng 2: N số — hoán vị P
Kết quả ra (NGHICHDAO.OUT):
  • Một số nguyên — số cặp nghịch thế

NGHICHDAO.INP

5
2 3 1 5 4

NGHICHDAO.OUT

3
Giải thích: Các cặp nghịch thế: (2,1), (3,1), (5,4).
📝 Code C++ — Merge Sort đếm nghịch thế 
#include <bits/stdc++.h>
using namespace std;
long long ans = 0;

void mergeSort(vector<int>& a, int l, int r) {
    if (l >= r) return;
    int mid = (l + r) / 2;
    mergeSort(a, l, mid);
    mergeSort(a, mid + 1, r);
    vector<int> tmp;
    int i = l, j = mid + 1;
    while (i <= mid && j <= r) {
        if (a[i] <= a[j]) tmp.push_back(a[i++]);
        else {
            ans += mid - i + 1; // tất cả a[i..mid] > a[j]
            tmp.push_back(a[j++]);
        }
    }
    while (i <= mid) tmp.push_back(a[i++]);
    while (j <= r) tmp.push_back(a[j++]);
    for (int k = l; k <= r; k++) a[k] = tmp[k - l];
}

int main() {
    ifstream in("NGHICHDAO.INP");
    ofstream out("NGHICHDAO.OUT");
    int n;
    in >> n;
    vector<int> a(n);
    for (int i = 0; i < n; i++) in >> a[i];
    mergeSort(a, 0, n - 1);
    out << ans;
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Dùng Merge Sort cải tiến hoặc BIT (C5). Cách Merge Sort: khi merge 2 nửa đã sort, nếu lấy phần tử a[j] từ nửa phải trước a[i] từ nửa trái, thì a[i..mid] đều > a[j] → cộng (mid − i + 1) nghịch thế. O(N log N). Duyệt 2 vòng for → O(N2) ≈ 2.5×1011 → TLE. Cách BIT: duyệt từ phải sang trái, dùng BIT đếm phần tử nhỏ hơn đã gặp (tương tự Bài 2).
Bài 7: CÂY KHUNG NHỎ NHẤT
SPANTREE.INP / SPANTREE.OUT

Cho đồ thị vô hướng liên thông N đỉnh, M cạnh có trọng số. Tìm tổng trọng số cây khung nhỏ nhất.

Dữ liệu vào (SPANTREE.INP):
  • Dòng 1: N, M (2 ≤ N ≤ 105; N−1 ≤ M ≤ 2×105)
  • M dòng tiếp: u, v, w (1 ≤ u, v ≤ N; 1 ≤ w ≤ 109)
Kết quả ra (SPANTREE.OUT):
  • Một số nguyên — tổng trọng số cây khung nhỏ nhất

SPANTREE.INP

4 5
1 2 1
2 3 2
3 4 3
1 3 4
2 4 5

SPANTREE.OUT

6
Giải thích: Chọn cạnh 1-2(1), 2-3(2), 3-4(3). Tổng = 6.
📝 Code C++ — Kruskal + DSU 
#include <bits/stdc++.h>
using namespace std;
int par[100005], rnk[100005];

int find(int x) {
    while (par[x] != x) x = par[x] = par[par[x]];
    return x;
}

bool unite(int a, int b) {
    a = find(a); b = find(b);
    if (a == b) return false;
    if (rnk[a] < rnk[b]) swap(a, b);
    par[b] = a;
    if (rnk[a] == rnk[b]) rnk[a]++;
    return true;
}

int main() {
    ifstream in("SPANTREE.INP");
    ofstream out("SPANTREE.OUT");
    int n, m;
    in >> n >> m;
    for (int i = 1; i <= n; i++) { par[i] = i; rnk[i] = 0; }
    vector<tuple<long long, int, int>> edges(m);
    for (int i = 0; i < m; i++) {
        int u, v; long long w;
        in >> u >> v >> w;
        edges[i] = {w, u, v};
    }
    sort(edges.begin(), edges.end());
    long long total = 0;
    int cnt = 0;
    for (int i = 0; i < (int)edges.size(); i++) {
        long long w = get<0>(edges[i]);
        int u = get<1>(edges[i]), v = get<2>(edges[i]);
        if (unite(u, v)) {
            total += w;
            if (++cnt == n - 1) break;
        }
    }
    out << total;
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Dùng DSU (C4) + Kruskal. Sort cạnh theo trọng số tăng. Duyệt từng cạnh: nếu 2 đầu khác thành phần liên thông (find khác nhau) → chọn cạnh + unite. Dừng khi đủ N−1 cạnh. DSU giúp kiểm tra/hợp nhất thành phần liên thông gần O(1). Tổng: O(M log M) cho sort + O(M × α(N)) cho DSU ≈ O(M log M).
Bài 8: LIÊN THÔNG ONLINE
LIENTHONG.INP / LIENTHONG.OUT

Cho N đỉnh, ban đầu không có cạnh nào. Có Q thao tác:

• 1 u v: Thêm cạnh (u, v).
• 2 u v: Hỏi u, v có liên thông không.
• 3: In số thành phần liên thông hiện tại.

Dữ liệu vào (LIENTHONG.INP):
  • Dòng 1: N, Q (1 ≤ N, Q ≤ 5×105)
  • Q dòng tiếp: mỗi dòng 1 thao tác
Kết quả ra (LIENTHONG.OUT):
  • Với thao tác 2: in "YES" hoặc "NO"
  • Với thao tác 3: in số thành phần liên thông

3
2 1 3
2 1 2

3
NO
YES
📝 Code C++ — DSU + đếm thành phần 
#include <bits/stdc++.h>
using namespace std;
const int MAXN = 5e5 + 5;
int par[MAXN], rnk[MAXN], comp;

int find(int x) {
    while (par[x] != x) x = par[x] = par[par[x]];
    return x;
}

void unite(int a, int b) {
    a = find(a); b = find(b);
    if (a == b) return;
    comp--;
    if (rnk[a] < rnk[b]) swap(a, b);
    par[b] = a;
    if (rnk[a] == rnk[b]) rnk[a]++;
}

int main() {
    ifstream in("LIENTHONG.INP");
    ofstream out("LIENTHONG.OUT");
    int n, q;
    in >> n >> q;
    comp = n;
    for (int i = 1; i <= n; i++) { par[i] = i; rnk[i] = 0; }
    while (q--) {
        int t;
        in >> t;
        if (t == 1) {
            int u, v; in >> u >> v;
            unite(u, v);
        } else if (t == 2) {
            int u, v; in >> u >> v;
            out << (find(u) == find(v) ? "YES" : "NO") << "\n";
        } else {
            out << comp << "\n";
        }
    }
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Dùng DSU (C4). Biến comp = số thành phần liên thông, ban đầu = N. Mỗi lần unite thành công (2 đỉnh khác nhóm): comp--. Thao tác 3 trả comp. Nếu dùng BFS/DFS mỗi truy vấn → O(N+M) mỗi lần → TLE. DSU: mỗi thao tác O(α(N)) ≈ O(1).
Bài 9: MAX CỬA SỔ TRƯỢT
SLIDING.INP / SLIDING.OUT

Cho dãy N số nguyên và số K. Với mỗi cửa sổ gồm K phần tử liên tiếp (từ vị trí i đến i+K−1), tìm giá trị lớn nhất trong cửa sổ đó.

Dữ liệu vào (SLIDING.INP):
  • Dòng 1: N, K (1 ≤ K ≤ N ≤ 106)
  • Dòng 2: N số nguyên ai (|ai| ≤ 109)
Kết quả ra (SLIDING.OUT):
  • N−K+1 số trên một dòng, mỗi số là max của cửa sổ tương ứng

SLIDING.INP

8 3
1 3 -1 -3 5 3 6 7

SLIDING.OUT

3 3 5 5 6 7
Giải thích: Cửa sổ [1,3,−1]→3, [3,−1,−3]→3, [−1,−3,5]→5, [−3,5,3]→5, [5,3,6]→6, [3,6,7]→7.
📝 Code C++ — Deque (hàng đợi hai đầu) 
#include <bits/stdc++.h>
using namespace std;

int main() {
    ifstream in("SLIDING.INP");
    ofstream out("SLIDING.OUT");
    int n, k;
    in >> n >> k;
    vector<int> a(n);
    for (int i = 0; i < n; i++) in >> a[i];
    deque<int> dq; // lưu chỉ số, a[dq] giảm dần
    for (int i = 0; i < n; i++) {
        // Loại phần tử ngoài cửa sổ
        while (!dq.empty() && dq.front() <= i - k)
            dq.pop_front();
        // Loại phần tử nhỏ hơn a[i] ở cuối (không bao giờ là max)
        while (!dq.empty() && a[dq.back()] <= a[i])
            dq.pop_back();
        dq.push_back(i);
        if (i >= k - 1)
            out << a[dq.front()] << " ";
    }
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Dùng Deque (C4). Cách thô: mỗi cửa sổ duyệt K phần tử tìm max → O(NK) ≈ 1012 → TLE. Tối ưu bằng deque: duy trì deque chứa chỉ số với giá trị giảm dần từ đầu đến cuối. Khi thêm a[i]: loại cuối deque nếu ≤ a[i] (chúng sẽ không bao giờ là max trong tương lai). Loại đầu deque nếu ngoài cửa sổ (chỉ số ≤ i − K). Max luôn ở đầu deque. Mỗi phần tử vào/ra deque đúng 1 lần → O(N). Mỗi phần tử vào/ra deque đúng 1 lần nên tổng O(N).
Bài 10: PHẦN TỬ LỚN HƠN GẦN NHẤT
PHANLON.INP / PHANLON.OUT

Cho dãy N số nguyên a1, a2, …, aN. Với mỗi phần tử ai, tìm chỉ số j nhỏ nhất sao cho j > i và aj > ai (phần tử lớn hơn gần nhất bên phải). Nếu không tồn tại, in −1.

Dữ liệu vào (PHANLON.INP):
  • Dòng 1: N (1 ≤ N ≤ 106)
  • Dòng 2: N số nguyên ai (|ai| ≤ 109)
Kết quả ra (PHANLON.OUT):
  • Một dòng gồm N số — mỗi số là chỉ số j hoặc −1

PHANLON.INP

5
4 2 1 5 3

PHANLON.OUT

4 4 4 -1 -1
Giải thích: a[1]=4 → a[4]=5 > 4, nên ans[1]=4. a[2]=2 → a[4]=5 > 2, ans[2]=4. a[3]=1 → a[4]=5 > 1, ans[3]=4. a[4]=5 → không có phần tử lớn hơn bên phải, ans[4]=−1. a[5]=3 → tương tự, ans[5]=−1.
📝 Code C++ — Stack 
#include <bits/stdc++.h>
using namespace std;

int main() {
    ifstream in("PHANLON.INP");
    ofstream out("PHANLON.OUT");
    int n;
    in >> n;
    vector<int> a(n + 1), ans(n + 1, -1);
    for (int i = 1; i <= n; i++) in >> a[i];
    stack<int> st; // lưu chỉ số, a[st] giảm dần từ đáy lên đỉnh
    for (int i = 1; i <= n; i++) {
        // a[i] lớn hơn phần tử trên đỉnh → đó là câu trả lời cho đỉnh
        while (!st.empty() && a[st.top()] < a[i]) {
            ans[st.top()] = i;
            st.pop();
        }
        st.push(i);
    }
    for (int i = 1; i <= n; i++)
        out << ans[i] << " \n"[i == n];
    in.close();
    out.close();

    return 0;
}
💡 Ý tưởng: Dùng Stack (C4). Cách thô: với mỗi i duyệt j = i+1..N tìm a[j] > a[i] → O(N2) ≈ 1012 → TLE. Tối ưu bằng stack: duyệt i từ trái sang phải, duy trì stack chứa chỉ số có a[] giảm dần. Khi gặp a[i] > a[đỉnh stack] → pop, ghi ans = i cho phần tử vừa pop (vì i là chỉ số đầu tiên bên phải lớn hơn nó). Mỗi phần tử vào/ra stack đúng 1 lần → O(N).
