Dấu hiệu trong đề Thuật toán cần dùng Ví dụ bài

2.2. Xác định độ phức tạp theo giới hạn N

Giới hạn N Độ phức tạp cho phép Thuật toán phù hợp
N ≤ 20 O(2^N), O(N!) Vét cạn, quay lui, bitmask
N ≤ 100 O(N³) Floyd, DP 3 chiều
N ≤ 1000 O(N²) DP 2 chiều, 2 vòng lặp
N ≤ 10⁵ O(N log N) Sort, Binary Search, BIT
N ≤ 10⁶ O(N) Two Pointers, Prefix Sum
N ≤ 10⁷ O(N) Sàng nguyên tố, duyệt đơn

3.2. TỔNG TIỀN TỐ 2D

// sum[i][j] = tổng hình chữ nhật từ (1,1) đến (i,j)
for (int i = 1; i <= m; i++) {
    for (int j = 1; j <= n; j++) {
        sum[i][j] = a[i][j] + sum[i-1][j] + sum[i][j-1] - sum[i-1][j-1];
    }
}

// Truy vấn tổng HCN từ (r1,c1) đến (r2,c2)
int tong = sum[r2][c2] - sum[r1-1][c2] - sum[r2][c1-1] + sum[r1-1][c1-1];

3.3. TWO POINTERS - SLIDING WINDOW

Áp dụng: Tìm đoạn con liên tiếp thỏa điều kiện (tổng ≤ K, max - min ≤ K,...) 
// Tìm đoạn con dài nhất có tổng ≤ K
long long sum = 0;
int L = 0, maxLen = 0;

for (int R = 0; R < n; R++) {
    sum += a[R];               // Mở rộng bên phải
    
    while (sum > K && L <= R) {  // Thu hẹp bên trái nếu vi phạm
        sum -= a[L];
        L++;
    }
    
    maxLen = max(maxLen, R - L + 1);  // Cập nhật đáp án
}

3.4. DSU (Union-Find) - GỘP NHÓM

int cha[MAXN];
int rank_set[MAXN];  // Để union by rank

void khoi_tao(int n) {
    for (int i = 1; i <= n; i++) {
        cha[i] = i;
        rank_set[i] = 0;
    }
}

int tim_goc(int v) {
    if (v == cha[v]) return v;
    return cha[v] = tim_goc(cha[v]);  // Path compression - RẤT QUAN TRỌNG!
}

void hop_nhat(int a, int b) {
    a = tim_goc(a);
    b = tim_goc(b);
    if (a != b) {
        if (rank_set[a] < rank_set[b]) swap(a, b);
        cha[b] = a;
        if (rank_set[a] == rank_set[b]) rank_set[a]++;
    }
}

3.5. BINARY INDEXED TREE (BIT)

int bit[MAXN];
int n;

void update(int idx, int delta) {
    for (; idx <= n; idx += idx & (-idx))
        bit[idx] += delta;
}

int query(int idx) {  // Tổng từ 1 đến idx
    int sum = 0;
    for (; idx > 0; idx -= idx & (-idx))
        sum += bit[idx];
    return sum;
}

// Tổng đoạn [l, r] = query(r) - query(l-1)

// f[j] = giá trị lớn nhất với trọng lượng tối đa j

3.8. SÀNG NGUYÊN TỐ ERATOSTHENES

bool laSoNguyenTo[MAX_VAL];
vector<int> dsSoNguyenTo;

void sangNguyenTo(int n) {
    fill(laSoNguyenTo, laSoNguyenTo + n + 1, true);
    laSoNguyenTo[0] = laSoNguyenTo[1] = false;
    
    for (int i = 2; i * i <= n; i++) {
        if (laSoNguyenTo[i]) {
            for (int j = i * i; j <= n; j += i) {
                laSoNguyenTo[j] = false;
            }
        }
    }
    
    for (int i = 2; i <= n; i++) {
        if (laSoNguyenTo[i]) dsSoNguyenTo.push_back(i);
    }
}

PHẦN 4: BẪY THƯỜNG GẶP & CÁCH TRÁNH

4.1. TRÀN SỐ (Overflow)

NGUY HIỂM NHẤT! Sai 50-70% test mà không biết tại sao.
Tình huống Giải pháp
Tổng của N số (N ≤ 10⁵, giá trị ≤ 10⁹) Dùng long long
Nhân 2 số lớn (a * b với a, b ≤ 10⁹) Ép kiểu: (long long)a * b
Prefix sum 2D Dùng long long cho mảng sum
Đếm số cách (kết quả rất lớn) Modulo theo yêu cầu đề
So sánh tích lớn (a*b với c*d) Dùng long double hoặc chia tránh nhân

5.1. Phân bổ thời gian (150 phút - 4 bài)

Giai đoạn Thời gian Công việc

Kiểm tra 10-15 phút Rà soát tất cả bài

5.2. Khi gặp bài khó - Chiến thuật ăn điểm từng phần

KHÔNG BỎ CUỘC! Luôn cố gắng lấy điểm subtask.
  1. Subtask N nhỏ (N ≤ 20): Vét cạn O(2^N) hoặc O(N²)
  2. Subtask trường hợp đặc biệt: Nếu tất cả bằng nhau, nếu đã sắp xếp,...
  3. Thuật toán tham lam: Không đúng 100% nhưng có thể được 30-50%
  4. Random: Đôi khi random + kiểm tra có thể ăn vài test
